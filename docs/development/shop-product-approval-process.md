# 🛒 상품 사전 검수(관리자 승인) 및 상태 관리 프로세스

본 문서는 FarmBalance 상점(Shop) 도메인의 상품 등록 시 부적절한 물품(장난식 등록 등)을 방지하기 위한 **관리자 사전 검수(Pre-Moderation) 시스템**의 전체 흐름과 엣지 케이스 처리 기준을 명세합니다.

---

## 1. 상품 상태(ProductStatus) 정의

| 상태값 | 화면 표시 (태그/뱃지) | 의미 및 노출 여부 |
| :--- | :--- | :--- |
| **`PENDING`** | 🟡 **검수중** | 판매자가 상품을 최초 등록했거나 수정한 상태. **장터 미노출** |
| **`ACTIVE`** | 🟢 **판매중** | 관리자가 승인 완료한 정상 상품. **장터 노출 및 구매 가능** |
| **`REJECTED`** | 🔴 **반려됨** | 관리자 심사 결과 부적절한 상품으로 판정됨. **장터 미노출** |
| **`INACTIVE`** | ⚪️ **숨김** | 승인된 상품을 판매자가 임의로 숨긴 상태. **장터 미노출** |
| **`SOLDOUT`** | 🟠 **품절** | 승인된 상품의 재고가 없거나 판매자가 강제로 품절 처리. **장터 노출(품절 표시, 구매 불가)** |

---

## 2. 관리자 승인 프로세스 전체 흐름

1. **상품 등록 및 관리자 인계**
   - 사용자가 [상품 등록]을 누르면, 상품은 즉시 `PENDING(검수중)` 상태로 데이터베이스에 저장되며 관리자 대시보드 심사 목록에 나타납니다.
   - ⚠️ `PENDING` 및 `REJECTED` 상태의 상품은 장터(Shop) 목록이나 검색 결과에 절대 노출되지 않습니다.

2. **판매자 화면 UI 표시**
   - 승인 전(`PENDING`): 판매 상품 관리 테이블의 상태 컬럼에 "검수중" 이라는 뱃지(태그)가 표시됩니다.
   - 승인 후(`ACTIVE` 등): "검수중" 태그가 사라지고, 대신 상태를 직접 변경할 수 있는 **[판매중 / 숨김 / 품절] 드롭다운 메뉴**가 활성화됩니다.

3. **관리자의 결정 (승인 vs 반려)**
   - **승인 시 (`ACTIVE`):** 상태가 변경되며 즉시 장터 목록에 노출됩니다.
   - **반려 시 (`REJECTED`):** 테이블 상태 컬럼에 "반려됨" 태그가 노출되며, 유저가 사유를 확인하고 다시 수정할 수 있도록 안내합니다.

4. **상품 삭제 로직 (`Soft Delete`)**
   - 상품 삭제 시 DB에서 완전히 지우지 않고 `products` 테이블의 `deleted_at` 컬럼을 현재 시간(`NOW()`)으로 업데이트합니다.
   - 관리자 검수 대기 목록은 `deleted_at IS NULL` 조건으로 조회하므로, 유저가 삭제하는 즉시 관리자의 검수 대기열에서 사라집니다.

## 3. 핵심 예외 및 엣지 케이스 처리 기준

### 3.1 수정(Edit) 권한 및 재검수 로직
* **검수 중 (`PENDING`) 에는 수정 불가:** 
  - 관리자의 심사 혼선을 막기 위해 `PENDING` 상태일 때는 판매자 테이블의 **[수정] 버튼을 비활성화(Disabled)** 합니다.
* **반려 (`REJECTED`) 된 상품의 수정:** 
  - 반려된 상품은 다시 내용을 보완해야 하므로 **[수정] 버튼이 활성화**됩니다.
  - 수정 완료 시 다시 `PENDING` 상태로 돌아가 관리자에게 재심사를 받습니다.
* **승인 (`ACTIVE`) 된 상품의 수정:** 
  - 판매 중인 상품을 수정할 수 있습니다 (버튼 활성화).
  - 단, 수정이 완료되는 즉시 다시 **`PENDING` 상태로 돌아가며 장터에서 즉시 내려갑니다.** (수정 후 재검수 원칙)

### 3.2 내용 수정(Update)과 단순 상태 변경(Patch Status)의 구별
* 상품의 제목, 가격, 내용, 이미지 등을 수정하는 것은 **재검수 대상(PENDING 전환)**입니다.
* 그러나, 이미 승인받은 상품에 대해 판매자가 단순히 드롭다운에서 **[판매중 ↔ 숨김 ↔ 품절] 상태만 변경하는 것은 재검수 대상이 아닙니다.** (상태 변경 전용 API 활용 시 PENDING으로 전환 금지)

### 3.3 장바구니/찜 담기 후 상태가 변한 경우의 방어 로직
* 누군가 장바구니에 상품을 담았는데, 이후 판매자가 상품을 수정해서 `PENDING`이 되거나 삭제(`deleted_at`)한 경우:
* 사용자가 해당 상품 상세페이지나 장바구니 결제 단계 진입 시, **"판매가 중단되거나 검수 중인 상품입니다"**라는 에러를 반환하고 튕겨내도록 백엔드 조회 로직에 락(Lock)을 걸어두어야 합니다.
