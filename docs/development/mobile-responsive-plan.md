## 1. 사용자가 제안한 항목 검토

### 1.1 모든 페이지 모바일 화면 대응 ✅
필수. 단, 페이지마다 난이도와 우선순위 다름 → **3단계로 분류** (아래 3장 참조).

### 1.2 메뉴 방식 (햄버거 vs 하단 탭바)
| 방식 | 장점 | 단점 | 적합한 곳 |
|---|---|---|---|
| **햄버거** | 메뉴 항목 수 무제한, 화면 차지 적음 | 한 번 탭 더 필요, 접근성 약함 | 관리자, 부가 메뉴 |
| **하단 탭바** | 한 손 조작, 발견성 높음, 모바일 표준 | 항목 4~5개로 제한, 화면 하단 점유 | 일반 사용자 메인 메뉴 |

**추천 — 하이브리드 방식:**
- 일반 사용자 페이지: **하단 탭바 (4개)** = 홈 / 쇼핑 / 커뮤니티 / 마이페이지
- 그 외 메뉴 (농장, 정책, balance, stores 등): **햄버거 메뉴**에 정리
- 관리자 페이지: **햄버거 메뉴**

### 1.3 사이즈 체크 ✅
필수. **공식 테스트 뷰포트 정하기** → 6장에 명시.

### 1.4 업로드 모바일 대응 ✅
필수. native input은 모바일에서도 동작하지만 **카메라 직접 촬영 / 갤러리 선택 / HEIC 변환 / 용량** 별도 고려 필요 → 4.4장에 정리.

## 2. 추가로 필요한 항목 (사용자가 빠뜨린 부분)

### 2.1 ★ 터치 타겟 사이즈
- 모든 클릭 가능 요소는 **최소 44×44px** (Apple HIG / WCAG 권장)
- 현재 버튼/링크 일부가 desktop 기준 작음 → 패딩 점검

### 2.2 ★ 데이터 테이블 모바일 패턴
관리자 페이지 다수가 가로 테이블. 모바일에서 3가지 옵션:
1. **카드 변환** (권장): 각 row를 카드로, label-value 세로 배치
2. **가로 스크롤**: 컴팩트 테이블 그대로 + `overflow-x: auto`
3. **컬럼 숨김**: 핵심 컬럼만 노출, 상세는 expand/모달

### 2.3 ★ 모달/다이얼로그
모바일에서는 **풀스크린 시트** 또는 **bottom sheet**가 표준. 현재 모달 6+개 모두 점검 필요.

### 2.4 ★ iOS 노치/홈 인디케이터 대응
- `env(safe-area-inset-bottom)` 사용 (특히 하단 탭바 채택 시)
- `viewport-fit=cover` meta 태그 확인

### 2.5 ★ 폼 UX
- input `font-size: 16px` 이상 (iOS 자동 줌 방지)
- 적절한 `inputmode`, `autocomplete`, `type` 지정 (tel, email, numeric 등)
- 폼 1열 배치, 라벨 위에

### 2.6 ★ 이미지 최적화
- `next/image` 사용 + `sizes` 속성으로 모바일에서 작은 이미지 로드
- 업로드 시 클라이언트 측 리사이즈 (특히 농장 사진, 상품 사진)

### 2.7 ★ 한글 줄바꿈
- `word-break: keep-all; overflow-wrap: break-word` — 한글이 어절 단위로 끊기게
- 긴 제목, 게시글 본문 등에 일괄 적용 권장

### 2.8 ★ 헤더 동작
- 스크롤 시 헤더 축소 또는 hide-on-scroll 고려 (특히 모바일 화면 세로 공간 절약)

### 2.9 ★ 지도 (Kakao Maps)
- 컨테이너 `height` 고정값 → 뷰포트 비율 기반(`60vh` 등)으로 변경
- 모바일에서 줌/팬 제스처가 페이지 스크롤과 충돌 안 하도록 설정

### 2.10 ★ 차트 (Recharts)
- `ResponsiveContainer` 잘 쓰고 있지만 **최소 높이** 보장 필요
- 모바일에서는 일부 차트 단순화 (예: pie chart 라벨 숨기고 legend만)

### 2.11 ★ 접근성 (a11y)
- 키보드 포커스 표시
- ARIA 라벨 (특히 햄버거/탭바 아이콘만 있는 버튼)
- 컬러 콘트라스트 4.5:1 이상

### 2.12 ★ 성능
- 모바일은 네트워크/CPU 약함
- 코드 스플리팅 점검, 큰 차트/지도 라이브러리는 dynamic import
- 폰트 로딩 최적화 (`font-display: swap`)

### 2.13 ★ 가로 스크롤 방지
- `<body>` overflow-x hidden 또는 모든 요소 max-width: 100% 점검
- viewport meta: `width=device-width, initial-scale=1`

### 2.14 ★ QA 체크리스트
- 실제 기기 테스트 (iPhone, Android 각 1대 이상)
- 가로/세로 모드 전환
- 키보드 올라왔을 때 UI 안 깨지는지

## 4. 구현 전략

### 4.1 브레이크포인트 표준화 (필수, 1순위)
`globals.css`에 변수 추가:
```css
:root {
  --bp-sm: 480px;   /* 작은 폰 */
  --bp-md: 768px;   /* 큰 폰 / 작은 태블릿 (현재 기준) */
  --bp-lg: 1024px;  /* 태블릿 / 작은 노트북 */
  --bp-xl: 1280px;  /* 데스크탑 */
}
```
미디어쿼리는 **모바일 우선(min-width) 또는 데스크탑 우선(max-width)** 중 하나로 통일.
> 현 코드가 `max-width: 768px` 위주이므로 **데스크탑 우선** 유지 권장.

### 4.2 공통 모바일 패턴 컴포넌트 신설
- `<MobileNav>` (하단 탭바)
- `<HamburgerMenu>` (햄버거 드로어)
- `<BottomSheet>` (모달 대체)
- `<ResponsiveTable>` (카드 변환 테이블)
- `<UploadField>` (모바일 친화 업로드: 카메라/갤러리 선택, 미리보기, 리사이즈)

### 4.3 글로벌 CSS 보강
```css
/* iOS 자동 줌 방지 */
input, select, textarea { font-size: 16px; }
/* 가로 스크롤 방지 */
html, body { overflow-x: hidden; }
/* 한글 줄바꿈 */
body { word-break: keep-all; overflow-wrap: break-word; }
/* 터치 타겟 최소 사이즈 */
button, a.btn { min-height: 44px; }
/* iOS safe area */
.bottom-tabbar { padding-bottom: env(safe-area-inset-bottom); }
```

### 4.4 업로드 모바일 대응 상세
- `<input accept="image/*" capture="environment">` → 카메라 직접 호출 가능
- 다중 업로드: `multiple` 속성
- 클라이언트 측 이미지 리사이즈 (예: 농장/상품 사진 max 2048px) — 모바일 데이터 절약
- HEIC → JPEG 변환 (iOS 17 이하 대응) — `heic2any` 라이브러리 검토
- 업로드 진행률 표시 (현재 spinner만)

### 4.5 admin 테이블 변환
- 768px 이하: 각 row → 카드 (제목 + key-value 그리드 + 액션 버튼)
- 또는 핵심 컬럼 2~3개만 표시 + tap to expand

### 4.6 지도/차트
- 지도: `height: 60vh; min-height: 320px;`
- 차트: 모바일에서 legend 위치 하단 변경, 라벨 축약

## 5. 작업 단계 제안 (Phase 별)

| Phase | 작업 | 예상 기간 | 산출물 |
|---|---|---|---|
| **0. 준비** | 브레이크포인트/공통 스타일 표준화, 공통 컴포넌트 신설 | 2~3일 | `globals.css` 보강, `<MobileNav>`, `<BottomSheet>` 등 |
| **1. 글로벌 레이아웃** | Header, Footer, 메인 메뉴 모바일화 (햄버거+하단탭) | 2일 | 모든 페이지에서 메뉴 정상 |
| **2. P0 핵심 페이지** | 홈/쇼핑/커뮤니티/마이페이지/로그인 | 4~5일 | 일반 사용자 핵심 동선 완성 |
| **3. P0 고난이도** | 농장 등록, 체크아웃, 커뮤니티 글쓰기 (업로드 포함) | 4~5일 | 업로드 컴포넌트 포함 |
| **4. P1** | 추천/balance/stores/gov (차트·지도) | 3~4일 | 차트·지도 페이지 완성 |
| **5. 관리자** | admin 테이블 모바일 변환, 대시보드 | 3~4일 | 관리자 페이지 |
| **6. QA / 폴리싱** | 실기기 테스트, a11y, 성능, 문구 줄바꿈 | 2~3일 | 출시 가능 상태 |

**총 예상**: 약 **3~4주** (1인 기준, 디자인 시안 별도 가정)

## 6. QA 체크리스트

### 6.1 필수 테스트 뷰포트
- **375 × 667** (iPhone SE)
- **390 × 844** (iPhone 14)
- **412 × 915** (Pixel / 일반 안드로이드)
- **768 × 1024** (iPad)
- **1024 × 768** (iPad 가로)
- **1280 × 800** (작은 노트북)

### 6.2 페이지별 체크 항목 (각 페이지마다)
- [ ] 가로 스크롤이 발생하지 않는가
- [ ] 모든 버튼/링크가 손가락으로 누를 수 있는 크기인가 (≥44px)
- [ ] 텍스트가 잘리거나 박스 밖으로 넘치지 않는가
- [ ] 입력 시 키보드가 올라와도 UI가 깨지지 않는가
- [ ] 이미지가 적절한 사이즈로 로드되는가
- [ ] 모달/드롭다운이 화면 안에 들어오는가
- [ ] 차트/지도가 정상 표시되는가
- [ ] 가로 모드(landscape) 전환 시 정상인가
- [ ] iOS / Android 양쪽 실기기에서 확인

## 7. 즉시 결정해야 할 사항

다음 결정이 되어야 작업 시작 가능:

1. **메뉴 방식 최종 결정** — 하이브리드(하단탭+햄버거) vs 햄버거 only
2. **mypage 사이드바** — 모바일에서 햄버거에 통합할지, 별도 페이지로 유지할지
3. **admin 테이블 처리** — 카드 변환 vs 가로 스크롤 (페이지마다 다를 수 있음)
4. **이미지 업로드 라이브러리** — heic2any 등 추가 의존성 허용 여부
5. **하단 탭바 메뉴 4개** — 홈/쇼핑/커뮤니티/마이페이지가 맞는지 (또는 농장 포함?)
6. **PWA 적용 여부** — 홈 화면 추가, 오프라인 지원 등 (선택사항)
