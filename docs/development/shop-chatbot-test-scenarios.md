# Shop 챗봇 전체 테스트 시나리오

이 문서는 Shop 도메인 챗봇 기능을 검증하기 위한 테스트 시나리오 모음입니다.
각 시나리오는 입력 → 기대 결과 순으로 정리되어 있으며, 실패하는 케이스를 발견하면 이슈로 보고해 주세요.

---

## 📋 사전 준비

### 테스트 계정
- 구매자 계정 1개 (장바구니/주문 테스트용)
- 판매자 계정 1개 (상품 관리/주문 처리 테스트용)
  - 등록된 상품: `배추`, `상추`, `토마토`, `딸기` (다양한 상태)
  - PENDING(검수중) 상태 상품 최소 1개 필요

### 테스트 환경
```bash
# 백엔드 + 프론트 + AI 서버 모두 실행
docker compose up --build -d
```

### 결과 표기 가이드
- ✅ 정상 동작
- ❌ 실패
- ⚠️ 부분 동작 (메모 추가)

## ① 구매자(Buyer) — 일반 쇼핑 흐름

### 1.1 상품 탐색

| # | 입력 | 기대 결과 | 결과 |
|---|---|---|---|
| 1 | "뭐 팔아?" / "메뉴 보여줘" | 판매량 TOP 상품 카드 5개 표시 | ⬜ |
| 2 | "사과 검색해줘" | 사과 키워드 매칭 상품 목록 | ⬜ |
| 3 | "상추 상세 보여줘" | 상추 상세 정보 + 장바구니 담기 액션 | ⬜ |

**중요**: 응답 본문에 `id=숫자` 같은 내부 식별자가 노출되지 않아야 함.

---

### 1.2 장바구니 ⭐ (이번에 많이 수정됨)

| # | 입력 | 기대 결과 | 결과 |
|---|---|---|---|
| 4 | "장바구니 보여줘" | 장바구니 목록 + 합계 표시 | ⬜ |
| 5 | "양파 5개 장바구니에 담아줘" | 담기 성공 토스트 + **헤더 뱃지 즉시 갱신** | ⬜ |
| 6 | "장바구니에 배 3개로 변경해줘" | 1글자 상품명 인식 → "제철 배 5kg" 수량 변경 | ⬜ |
| 7 | "장바구니에서 양파 빼줘" | 양파 삭제 + 토스트 + 헤더 뱃지 즉시 갱신 | ⬜ |

**중요 확인**:
- 챗봇으로 장바구니 변경 시 `/shop/cart` 페이지 새로고침 없이 즉시 화면 갱신
- 헤더 장바구니 뱃지 카운트도 즉시 변경
- **결제 완료 후**: 주문한 상품이 장바구니에서 자동 삭제됨

## ⑤ 백엔드 API 직접 테스트 (선택)

```bash
# JWT는 로그인 후 쿠키나 응답에서 추출
TOKEN="<your-jwt>"
BASE="http://localhost:8080"
```

| # | 엔드포인트 | 시나리오 | 기대 결과 | 결과 |
|---|---|---|---|---|
| 55 | `PATCH /api/shop/seller/{id}/inventory` | PENDING 상품 가격·재고 변경 | 상태 PENDING 유지 | ⬜ |
| 56 | `PATCH /api/shop/seller/{id}` | PENDING 상품 콘텐츠 변경 시도 | `E-SHOP-007` 에러 | ⬜ |
| 57 | `POST /api/shop/order` | 주문 생성 | 주문된 상품의 cart item 자동 삭제 확인 | ⬜ |

예시:
```bash
# 가격·재고만 수정 (재검수 없음)
curl -X PATCH $BASE/api/shop/seller/1/inventory \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"price": 5000, "stock": 10}'

# PENDING 상품 콘텐츠 수정 시도 → E-SHOP-007 응답 확인
curl -X PATCH $BASE/api/shop/seller/1 \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"name":"새이름","price":5000,"stock":10,"description":"변경","categoryName":"채소","imageUrls":[]}'
```

## ⚠️ 알려진 한계 / 추후 개선

### 1. 다중 턴 컨텍스트 — 도메인 무관 답변
- 시나리오: PendingIntent 대기 중에 사용자가 "오늘 날씨 어때?" 입력
- 현재 동작: 슬롯 채우기 시도 후 실패 → 같은 슬롯 재질문
- 권장: 명시적 취소 키워드("아니야", "관둘게") 처리 필요

### 2. CLARIFY 옵션 클릭 시 후속 처리
- 챗봇이 "어떤 상품인지 선택해 주세요" 옵션 제시 후, 옵션 클릭이 원래 의도로 다시 라우팅되는지 확인 필요

### 3. 챗봇 응답 본문에 액션 JSON 노출 회귀
- 일부 케이스에서 `[ACTION:{...}]` 토큰이 본문에 그대로 보이는 회귀가 없는지 점검
