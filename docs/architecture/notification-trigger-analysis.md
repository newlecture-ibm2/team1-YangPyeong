# 🔔 알림 트리거 포인트 전체 분석

프로젝트 스펙(`project-spec.md`) + 실제 백엔드 서비스 코드를 기반으로, 사용자에게 알림이 가야 하는 모든 경우를 도메인별로 정리했습니다.

---

## 현재 상태 요약

| 항목 | 상태 |
|------|------|
| `NotificationService.createNotification()` | ✅ 구현됨 |
| `NotificationService.createBulkNotifications()` | ✅ 구현됨 |
| **실제로 호출하는 곳** | ❌ **없음** (전부 연결 필요) |

## 1. 🟢 수급(Balance) 도메인 — `BALANCE_WARN`

| # | 트리거 조건 | 수신 대상 | 메시지 예시 |
|---|------------|-----------|------------|
| B-1 | 수급 비율 > 150% (과잉경고) | 해당 작물 재배 중인 **농부** | "배추의 공급률이 165%로 과잉경고입니다. 대안 작물을 확인하세요." |
| B-2 | 수급 비율 120~150% (과잉주의) | 해당 작물 재배 중인 **농부** | "배추의 공급률이 135%로 과잉주의입니다." |
| B-3 | 수급 비율 50~80% (부족주의) | 해당 지역 **전체 농부** | "배추의 공급률이 65%로 부족주의입니다. 재배를 권장합니다." |
| B-4 | 수급 비율 < 50% (부족경고) | 해당 지역 **전체 농부** | "배추의 공급률이 40%로 부족경고입니다. 적극 재배를 권장합니다." |
| B-5 | 모든 경고(B-1~B-4) | **지자체(GOV)** 역할 사용자 | 동일 메시지 |

> **연결 위치**: `SupplyService.recalculate()` 또는 `calculateSupplyRatio()` 에서 상태 변경 감지 시

---

## 2. 🛒 주문(Shop/Order) 도메인 — `ORDER`

현재 이메일만 발송 중이며, **인앱 알림은 미구현** 상태입니다.

| # | 트리거 조건 | 수신 대상 | 메시지 예시 |
|---|------------|-----------|------------|
| O-1 | 새 주문 접수 | **셀러(판매자)** | "새 주문이 접수되었습니다. (주문번호: ORD-XXXX)" |
| O-2 | 주문 접수 확인 (ORDERED → ACCEPTED) | **구매자** | "주문하신 상품이 접수되었습니다." |
| O-3 | 배송 시작 (ACCEPTED → SHIPPED) | **구매자** | "주문하신 상품이 배송 시작되었습니다. (송장: FARM-XXXX)" |
| O-4 | 주문 거절/취소 (→ CANCELLED) | **구매자** | "주문이 거절되었습니다. (주문번호: ORD-XXXX)" |

> **연결 위치**: `OrderService.createOrder()`, `advanceOrderStatus()`, `shipOrder()`, `cancelOrder()`
> ⚠️ 현재 이메일(`sendStatusEmail`)과 **병행**하여 인앱 알림 추가 필요

## 📊 종합 — 우선순위별 구현 매트릭스

| 우선순위 | 도메인 | 트리거 수 | 난이도 | 비고 |
|:--------:|--------|:---------:|:------:|------|
| **P0** | 주문(Order) | 4건 | ⭐ 낮음 | 이미 이메일 발송 로직이 있어 병행 추가만 하면 됨 |
| **P0** | 농장 승인/반려 | 2건 | ⭐ 낮음 | `approve()`/`reject()` 마지막에 1줄 추가 |
| **P1** | 수급(Balance) | 5건 | ⭐⭐ 보통 | 상태 변경 감지 로직 추가 필요 |
| **P1** | 커뮤니티(댓글/채택) | 2건 | ⭐ 낮음 | `createComment()`/`acceptComment()` 끝에 추가 |
| **P2** | 정책 매칭 결과 | 1건 | ⭐ 낮음 | 정책 매칭 응답 시 알림 동시 생성 |
| **P2** | 지자체 권고 발송 | 1건 | ⭐⭐ 보통 | 별도 API 필요할 수 있음 |
| **P3** | 관리자 조치 | 3건 | ⭐ 낮음 | 관리자 서비스에 추가 |

> **총 18건의 알림 트리거 포인트**가 확인되었습니다.
