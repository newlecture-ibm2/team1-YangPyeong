# 🤖 Shop/Seller AI 자동화 설계 문서

> FarmBalance 상점 도메인에 AI를 적용하여 사용자 입력을 최소화하고 효율성을 극대화하는 방안

---

## 1. 현재 사용자 입력 분석

### 1.1 판매자 (Seller)

| 기능          | 수동 입력 항목                                                    | 고통 포인트                                   |
| ------------- | ----------------------------------------------------------------- | --------------------------------------------- |
| **상품 등록** | 상품명, 가격, 재고, 카테고리, 설명(10자~5000자), 이미지(최대 5장) | 설명 작성이 가장 어려움, 가격 결정이 막막함   |
| **상품 수정** | 위와 동일                                                         | 설명만 수정하고 싶어도 전체 폼을 다시 봐야 함 |
| **주문 관리** | 접수확인/거절 클릭                                                | 반복 작업, 거절 사유 작성 부담                |

### 1.2 구매자 (Buyer)

| 기능          | 수동 입력 항목                   | 고통 포인트               |
| ------------- | -------------------------------- | ------------------------- |
| **상품 탐색** | 검색 키워드, 카테고리 필터, 정렬 | 어떤 농산물이 좋은지 모름 |
| **주문/결제** | 수령인, 연락처, 주소, 배송메모   | 매번 같은 정보 반복 입력  |

## 2. AI 자동화 제안 — 우선순위별 정리

### ⭐ Tier 1: 즉시 적용 가능 (높은 효과, 낮은 난이도)

#### 2.1 🖼️ AI 상품 설명 자동 생성

**대상**: 판매자 — 상품 등록/수정 페이지

```
[현재] 판매자가 직접 설명 작성 (10자 이상, 최대 5000자)
[개선] 상품명 + 카테고리 + 이미지 → AI가 매력적인 설명 초안 생성
```

| 항목          | 내용                                                      |
| ------------- | --------------------------------------------------------- |
| **트리거**    | 상품명, 카테고리 입력 후 "✨ AI 설명 생성" 버튼 클릭       |
| **입력**      | `{ name: "유기농 상추 500g", category: "채소류" }`        |
| **AI 처리**   | LLM에게 "양평군 로컬 농산물 상품 설명 작성" 프롬프트 전달 |
| **출력**      | 매력적인 상품 설명 초안 (판매자가 수정 가능)              |
| **기대 효과** | 설명 작성 시간 **90% 단축**, 설명 품질 향상               |

**생성 예시:**

```
입력: "유기농 상추 500g" + "채소류"

AI 생성 결과:
"양평군의 맑은 물과 깨끗한 토양에서 유기농으로 재배한 신선한 상추입니다.
아삭한 식감과 풍부한 영양소가 특징이며, 수확 당일 발송하여
최상의 신선도를 보장합니다. 쌈밥, 샐러드, 비빔밥 등 다양한 요리에 활용해 보세요."
```

---

#### 2.2 💰 AI 적정 가격 추천

**대상**: 판매자 — 상품 등록/수정 페이지

```
[현재] 판매자가 가격을 직접 입력 (시세 모름)
[개선] 카테고리 + 상품명 기반으로 시장 적정 가격 범위 제안
```

| 항목            | 내용                                                          |
| --------------- | ------------------------------------------------------------- |
| **트리거**      | 상품명 + 카테고리 입력 시 자동 표시                           |
| **데이터 소스** | 같은 카테고리 상품들의 평균/중간/최빈 가격, KOSIS 시세 데이터 |
| **출력**        | 가격 입력란 옆에 `💡 시세: ₩7,000~₩9,000` 힌트 표시            |
| **기대 효과**   | 경쟁력 있는 가격 설정 가능, 가격 고민 시간 단축               |

---

#### 2.3 🏷️ AI 카테고리 자동 추천

**대상**: 판매자 — 상품 등록 페이지

```
[현재] 판매자가 드롭다운에서 수동 선택
[개선] 상품명 입력 시 AI가 자동으로 카테고리 추천
```

| 항목          | 내용                                                       |
| ------------- | ---------------------------------------------------------- |
| **트리거**    | 상품명 입력 완료 (blur 이벤트)                             |
| **처리**      | 상품명 키워드 분석 → 카테고리 매칭                         |
| **출력**      | 드롭다운에 추천 카테고리 자동 선택 + "🤖 AI 추천" 뱃지 표시 |
| **기대 효과** | 카테고리 선택 스텝 제거                                    |

```
"유기농 상추 500g" 입력 → 🤖 "채소류" 자동 선택
"양평 꿀 1kg" 입력     → 🤖 "가공식품" 자동 선택
```

---

### ⭐ Tier 2: 중간 난이도 (높은 효과)

#### 2.4 🔍 AI 맞춤 상품 추천 (구매자)

**대상**: 구매자 — 상점 메인 페이지

```
[현재] 전체 상품을 카테고리/정렬로만 탐색
[개선] 사용자의 농장 데이터 + 구매 이력 → 맞춤 상품 추천
```

| 항목       | 내용                                         |
| ---------- | -------------------------------------------- |
| **데이터** | 사용자 농장 작물 정보, 최근 구매 이력, 계절  |
| **출력**   | "000님을 위한 추천 상품" 섹션                |
| **예시**   | 고추 재배 농가 → 고추용 비료, 방충망 등 추천 |

---

#### 2.5 📊 판매자 인사이트 대시보드

**대상**: 판매자 — 주문 관리 페이지

```
[현재] KPI 카드에 단순 숫자만 표시
[개선] AI가 판매 패턴 분석 + 조언 제공
```

| 항목          | 내용                                                                       |
| ------------- | -------------------------------------------------------------------------- |
| **분석**      | 주문 빈도, 인기 상품, 시간대별 패턴, 재고 소진 예측                        |
| **출력 예시** | `🔔 "유기농 상추"의 재고가 3일 내 소진 예상입니다. 재고 추가를 권장합니다.` |
| **출력 예시** | `📈 이번 주 주문량이 지난 주 대비 25% 증가했습니다.`                        |

---

#### 2.6 📝 자동 거절 사유 생성

**대상**: 판매자 — 주문 거절 시

```
[현재] 거절 버튼만 클릭 (사유 없음)
[개선] 거절 시 AI가 정중한 거절 메시지 자동 작성 → 구매자 이메일에 포함
```

---

### ⭐ Tier 3: 장기 확장

#### 2.7 🤖 이미지 기반 원클릭 상품 등록

**대상**: 판매자 — 상품 등록

```
[현재] 5개 필드 + 이미지 수동 입력
[개선] 이미지만 올리면 AI가 나머지 전부 자동 채움
```

```
📷 이미지 업로드 → AI Vision 분석
   ↓
🏷️ 상품명: "친환경 유기농 방울토마토 500g" (자동)
📁 카테고리: "과일류" (자동)
💰 가격: ₩8,500 (시세 기반 자동)
📦 재고: 직접 입력 (이것만 수동)
📝 설명: "양평에서 재배한 당도 높은..." (자동)
```

---

#### 2.8 💬 AI 쇼핑 어시스턴트 (챗봇)

**대상**: 구매자

```
사용자: "이번 주 양평에서 나온 신선한 채소 뭐 있어?"
AI: "현재 양평 직거래 상품 중 채소류는 3개입니다:
     1. 유기농 상추 500g — ₩8,000 (재고 45개)
     2. 무농약 오이 1kg — ₩6,500 (재고 30개)
     장바구니에 담아드릴까요?"
```

### 📦 Tier 4: 주문 및 커뮤니티(장터) AI 확장

#### 2.9 🛒 AI 재주문(구독) 알림 매니저

**대상**: 구매자 — 장바구니/주문

```
[현재] 구매자가 매번 필요할 때 직접 상점에 들어와서 검색 후 구매
[개선] AI가 구매자의 과거 주문 이력(orders, order_items)을 분석해 소모 주기를 파악하고 적시에 알림 발송
```
> **예시**: "지난번에 구매하신 '양평 친환경 유정란 30구'가 떨어질 때가 되지 않았나요? 바로 구매하시겠어요?"

#### 2.10 🔍 AI 자연어 상품 검색 (Semantic Search)

**대상**: 구매자 — 상점(장터) 메인/검색

```
[현재] 정확한 상품명이나 카테고리(예: "배추", "토마토")를 검색해야만 상품이 나옴
[개선] AI가 유저의 일상적인 질문이나 의도를 파악하여, 상품의 설명(description)과 매칭해 가장 적합한 상품을 추천
```
> **예시**: 
> 유저 검색어: "비 오는 날 부침개 해먹기 좋은 채소 찾아줘"
> AI 검색 결과: "애호박", "부추", "감자" 관련 상품 우선 노출

#### 2.11 💌 AI 스마트 배송 메모 추천

**대상**: 구매자 / 배송 기사

```
[현재] 구매 시 배송 메모를 매번 수동으로 입력해야 함
[개선] 결제 단계에서 장바구니에 담긴 품목(예: 신선 채소 vs 잘 깨지는 계란)의 특성을 AI가 분석하여, 가장 적합한 맞춤형 배송 메모를 자동 제안
```
> **예시**: (장바구니에 유정란이 있을 때) "신선식품(계란)이 포함되어 있습니다. '파손주의! 부재 시 경비실에 맡겨주세요' 라고 메모를 남길까요?"

## 3. 구현 아키텍처

### 3.1 데이터 흐름

```
┌──── Frontend ─────────────────────────────────┐
│                                                │
│  상품 등록 폼                                   │
│  ┌──────────────────────────────────┐          │
│  │ 상품명: [유기농 상추 500g     ]   │          │
│  │ 카테고리: 🤖 채소류 (AI 추천)     │          │
│  │ 가격: [8,000] 💡 시세: 7,000~9,000│         │
│  │ 설명: [✨ AI 설명 생성] 버튼       │          │
│  └──────────────────────────────────┘          │
│         │ POST /api/ai/product-assist          │
│         ▼                                      │
│  BFF Route Handler (Next.js)                   │
└────────┼───────────────────────────────────────┘
         │ HTTP
         ▼
┌──── Backend (Spring Boot) ─────────────────────┐
│  shop/adapter/out/external/AiProductAdapter     │
│         │ HTTP                                  │
└─────────┼──────────────────────────────────────┘
          ▼
┌──── AI Server (FastAPI) ───────────────────────┐
│  routers/product_assist.py                      │
│      ↓                                          │
│  services/product_assist_service.py             │
│      ↓                                          │
│  llm/ (Gemini/Groq) → 설명 생성, 카테고리 추천    │
│  DB 조회 → 가격 범위 계산                         │
└─────────────────────────────────────────────────┘
```

### 3.2 AI Server 추가 파일

```
ai/app/
├── routers/product_assist.py      ← API 엔드포인트
├── services/product_assist_service.py  ← 비즈니스 로직
└── models/product_assist.py       ← Request/Response 스키마
```

### 3.3 Backend 추가 파일

```
com.farmbalance.shop/
└── adapter/out/external/
    └── AiProductAdapter.java      ← AI 서버 HTTP 호출
```

### 3.4 Frontend 추가 파일

```
frontend/
├── app/api/ai/product-assist/route.ts  ← BFF Route Handler
└── app/(main)/mypage/seller/
    └── register/
        └── useAiAssist.ts              ← AI 도우미 Hook
```

## 4. API 스펙 (초안)

### 4.1 상품 설명 생성

```
POST /api/ai/product-assist/description

Request:
{
  "productName": "유기농 상추 500g",
  "categoryName": "채소류"
}

Response:
{
  "description": "양평군의 맑은 물과 깨끗한 토양에서...",
  "generatedAt": "2026-05-04T20:30:00"
}
```

### 4.2 카테고리 추천

```
POST /api/ai/product-assist/category

Request:
{
  "productName": "양평 꿀 1kg"
}

Response:
{
  "recommendedCategory": "가공식품",
  "confidence": 0.92
}
```

### 4.3 적정 가격 추천

```
POST /api/ai/product-assist/price

Request:
{
  "productName": "유기농 상추 500g",
  "categoryName": "채소류"
}

Response:
{
  "minPrice": 7000,
  "maxPrice": 9000,
  "avgPrice": 8000,
  "dataSource": "플랫폼 내 동일 카테고리 상품 기준"
}
```
