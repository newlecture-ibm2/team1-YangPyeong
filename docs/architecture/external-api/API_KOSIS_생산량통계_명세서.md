## 1. 개요

**KOSIS(국가통계포털)** API는 통계청이 제공하는 연도별·지역별 농작물 생산량 공식 통계 데이터를 제공한다.  
FarmBalance 시스템에서 이 데이터는 **수급 밸런스 계산의 기초 입력값**으로 사용된다.

### 1.1 시스템 내 활용처

| 활용처 | 기능 ID | 구체적 활용 |
| :--- | :---: | :--- |
| **수급 밸런스 대시보드** | FRM-001 | 연도별 생산량 추이 그래프 (X: 연도, Y: 생산량) |
| **수요 예측 보정** | FRM-001 | 과거 5~10년 생산량 평균으로 `balance_data.supply_forecast` 기초값 산출 |
| **AI 수익성 점수** | FRM-002 | 작물별 생산량 트렌드(증가/감소) 분석 → 수익성 점수(25%) 보정 |
| **관리자 관제** | ADM-001 | 전국/지역별 주산지 히트맵 및 작물별 점유율 분석 |
| **지자체 정책 근거** | GOV-002 | 지역 농업 현황 리포트 생성 시 공식 통계 인용 |

### 1.2 다른 API와의 차별점

| 구분 | 기상청 / 흙토람 등 | KOSIS |
| :--- | :--- | :--- |
| **데이터 갱신 주기** | 실시간 ~ 일 단위 | **연 1회** (연말 또는 익년 초) |
| **호출 전략** | 실시간 호출 + Redis 캐시 | **DB 저장 + 연 1회 배치 갱신** |
| **파라미터 특성** | 좌표/주소 기반 | **통계 코드 기반** (코드 매핑 필수) |
| **데이터 형태** | 단건 응답 | **수년치 대량 데이터 일괄 응답** |

### 2.2 요청 파라미터

## 3. 응답 데이터 (Response)

### 3.1 주요 응답 필드

KOSIS 응답은 **수년치 데이터가 배열(Array)로 한꺼번에** 제공된다.

| 필드 | 타입 | 설명 | 활용 |
| :--- | :---: | :--- | :--- |
| `PRD_DE` | String | 통계 기준 시점 (예: `"2023"`) | 차트 X축 (연도) |
| `DT` | String | 통계 수치 (예: `"435.3053"`) | 차트 Y축 (생산량). **문자열 → 숫자 변환 필요** |
| `UNIT_NM` | String | 단위 (예: `"ha"`, `"M/T"`) | 차트 Y축 라벨 및 단위 표시 |
| `ITM_ID` | String | 항목 코드 (예: `"T76"`) | 작물+측정항목 식별자 |
| `ITM_NM` | String | 항목명 (예: `"양파:면적"`) | **작물명 파싱** — ":" 앞부분 = 작물명 |
| `C1` | String | 지역 코드 (예: `"31"`) | DB `region_code` 매핑 |
| `C1_NM` | String | 지역명 (예: `"경기도"`) | DB `region_name` 저장 |
| `C1_NM_ENG` | String | 지역 영문명 (예: `"Gyeonggi-do"`) | 다국어 지원 시 활용 |
| `TBL_ID` | String | 통계표 ID (예: `"DT_1ET0291"`) | 데이터 출처 추적 |
| `TBL_NM` | String | 통계표명 (예: `"채소생산량(조미채소)"`) | 데이터 출처 표시 |
| `ORG_ID` | String | 기관 코드 (예: `"101"`) | 통계청 식별 |

> ⚠️ **주의:** KOSIS는 `C2`(작물코드), `C2_NM`(작물명) 필드를 **제공하지 않는다.**  
> 작물 구분은 `ITM_ID`(항목코드)로 이루어지며, 작물명은 `ITM_NM`에서 파싱해야 한다.

### 4.2 수급 밸런스 계산과의 연동

`전체_통합.md`에 정의된 수급 계산 공식에서 KOSIS 데이터가 기여하는 부분:

```
공급 예측 = Σ(등록 면적 × 작물별 단위수확량) + 외부 공급 추정
                                                    ↑
                                          KOSIS 10년 평균 생산량 기반 추정

수요 예측 = 지역 인구 × 1인당 소비량 × 계절 계수
                        ↑
              KOSIS 소비량 통계 활용 가능
```

**구체적 활용:**

| 수급 공식 항목 | KOSIS 기여 | 설명 |
| :--- | :--- | :--- |
| `외부 공급 추정` | 10년 평균 생산량 | 양평 외 지역에서 유입되는 공급량 추정 근거 |
| `작물별 단위수확량` 검증 | 실제 통계 대조 | `crops.yield_per_sqm` 값의 정확성 검증용 |
| 생산량 트렌드 | 증감 추세 분석 | AI 수익성 점수(25%) 보정 — 생산량 감소 작물 = 희소가치 상승 |

### 4.3 결측치(Missing Data) 처리 정책

| 상황 | 처리 | UI 표시 |
| :--- | :--- | :--- |
| 특정 연도 데이터 없음 | `null`로 저장 | 차트에서 해당 연도 **점선으로 연결** (데이터 없음 표시) |
| 특정 지역 데이터 없음 | 상위 지역(도 단위) 데이터로 대체 | "※ 시군구 데이터 미제공, 도 단위 통계 사용" 안내 |
| 특정 작물 데이터 없음 | 유사 작물군 평균으로 추정 | "※ 추정치" 라벨 표시 |
| 당해 연도 미발표 | 전년도 데이터 + 추세 기반 예측 | "※ 예측값" 라벨 표시 |

## 5. ERD 연동 상세

### 5.1 `balance_data` 테이블 연동

| ERD 컬럼 | KOSIS 데이터 기여 | 설명 |
| :--- | :--- | :--- |
| `supply_forecast` | 10년 평균 생산량 기반 외부 공급 추정치 반영 | 내부 파종 등록 + KOSIS 외부 공급 = 총 공급 예측 |
| `demand_forecast` | KOSIS 소비 통계 참조 | 지역 인구 × 1인당 소비량 산출 근거 |
| `supply_ratio` | 산출된 공급/수요로 자동 계산 | 수급 비율(%) |
| `year` / `season` | KOSIS `PRD_DE`와 매핑 | 연도별 비교 분석 |

### 5.2 `crop_production_stats` 테이블 연동

| ERD 테이블 | 역할 |
| :--- | :--- |
| `crop_production_stats` | 연도별 재배면적·단수·생산량 저장 (독립 테이블, FK 없음) |

## 8. 예외 처리

### 8.1 에러 코드

| ErrorCode | HTTP | 상황 | 대응 |
| :--- | :---: | :--- | :--- |
| `API_CONNECTION_FAIL` | 502 | KOSIS 서버 무응답 | Retry 3회 → DB 기존 데이터 사용 |
| `STAT_DATA_NOT_FOUND` | 200 | 해당 작물/지역 통계 없음 | 상위 분류 또는 유사 작물 통계로 대체 |
| `STAT_CODE_MISMATCH` | 500 | 매핑 테이블의 코드가 KOSIS 변경으로 무효화 | 관리자 알림 + 이전 캐시 데이터 유지 |
| `BATCH_TIMEOUT` | 504 | 배치 수집 시 타임아웃 | 작물별 분할 수집으로 재시도 |

### 8.2 Fallback 전략

| 장애 상황 | Fallback |
| :--- | :--- |
| KOSIS API 장애 | DB에 저장된 이전 배치 데이터 사용 (최신 연도만 누락) |
| 매핑 코드 변경 | 관리자에게 알림 발송 + 기존 데이터 유지 |
| 당해 연도 미발표 | 전년도 값 + 최근 3년 추세선 기반 예측치 생성 |

### 8.3 Retry 정책

```
최대 재시도: 3회
대기 전략: Exponential Backoff (2초 → 4초 → 8초)
타임아웃: 요청당 10초 (대량 데이터 응답 고려)
```
