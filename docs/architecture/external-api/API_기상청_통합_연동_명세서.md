## 1. 개요

FarmBalance 시스템은 기상청이 제공하는 **2종의 공공 API**를 연동한다.

| 구분 | API명 | 역할 | 데이터 성격 |
| :---: | :--- | :--- | :--- |
| **API-A** | 단기예보 조회 서비스 | 오늘·내일의 실시간 기상 정보 제공 | 실시간 (3시간 주기 갱신) |
| **API-B** | 종관기상관측(ASOS) 일별 조회 | 과거 10년간의 기후 통계 제공 | 과거 기록 (배치 수집) |

### 1.1 시스템 내 활용처

```
                         ┌─────────────────────────────────────────┐
                         │          AI 작물 추천 엔진 (FRM-002)     │
                         │                                         │
  [API-A] 단기예보 ─────→│  환경 적합도 점수 (25%)                  │
                         │  = 10년 평년 기후 vs 작물 적정 조건 매칭   │
  [API-B] ASOS 10년 ───→│  + 현재 기상 이상 여부 보정               │
                         │                                         │
                         └─────────────────────────────────────────┘

  [API-A] 단기예보 ─────→ 영농 가이드 (관수·방제·시설 관리 알림)
                   ─────→ 재난 알림 (냉해·강풍·폭우 위험 푸시)
```

## 2. API-A: 단기예보 조회 서비스 (실시간)

### 2.1 호출 정보

| 항목 | 내용 |
| :--- | :--- |
| **Base URL** | `http://apis.data.go.kr/1360000/VilageFcstInfoService_2.0/getVilageFcst` |
| **Method** | `GET` |
| **제공처** | 공공데이터포털 (기상청) |

### 2.2 요청 파라미터 (Request)

| 항목명 | 변수명 | 필수 | 값(예시) | 설명 |
| :--- | :--- | :---: | :--- | :--- |
| 서비스키 | `serviceKey` | Y | `(발급키)` | 공공데이터포털에서 발급받은 **Encoded** 키 |
| 페이지번호 | `pageNo` | Y | `1` | |
| 한 페이지 결과 수 | `numOfRows` | Y | `1000` | 모든 예보 항목 수신을 위해 충분히 설정 |
| 응답형식 | `dataType` | N | `JSON` | **기본값 XML** → 반드시 JSON 명시 |
| 발표일자 | `base_date` | Y | `20260423` | YYYYMMDD |
| 발표시각 | `base_time` | Y | `0500` | 하단 '발표 시각 결정 로직' 참고 |
| 격자 X좌표 | `nx` | Y | `55` | 기상청 전용 격자 좌표 (**위경도 아님**) |
| 격자 Y좌표 | `ny` | Y | `127` | 기상청 전용 격자 좌표 |

> ⚠️ **핵심 주의:** 기상청 API는 GPS 위경도가 아닌 **자체 격자 좌표(nx, ny)**를 사용한다.  
> 백엔드 `infra/external-api` 계층에 **위경도 → 격자 좌표 변환 유틸리티**를 반드시 구현해야 한다.

### 2.3 응답 데이터 (Response) — 사용 항목만 정의

기상청 응답은 `response.body.items.item[]` 배열 안에 여러 카테고리가 혼합되어 온다.  
아래 6개 항목만 필터링하여 사용한다.

| 코드 | 항목명 | 단위 | FarmBalance 활용 로직 |
| :---: | :--- | :---: | :--- |
| `TMP` | 1시간 기온 | ℃ | 작물별 생육 적정 온도 체크, 냉해·고온 경보 트리거 |
| `REH` | 습도 | % | 고온다습(기온 30℃+ & 습도 80%+) 시 병해충 주의 알림 |
| `PCP` | 1시간 강수량 | mm | 강수 시 관수(물 주기) 불필요 판정. **문자열 파싱 필요** (하단 참고) |
| `PTY` | 강수 형태 | 코드 | 0: 없음, 1: 비, 2: 비/눈, 3: 눈, 4: 소나기. 눈 시 하우스 하중 관리 알림 |
| `SKY` | 하늘 상태 | 코드 | 1: 맑음, 3: 구름많음, 4: 흐림. 일조량 기반 작물 관리 판단 |
| `WSD` | 풍속 | m/s | **4m/s 이상** 시 약제 살포 자제 알림, 시설물 고정 권고 |

### 2.4 발표 시각(base_time) 결정 로직

단기예보는 하루 8회 발표되며, 데이터 생성에 **약 10~15분**이 소요된다.

| 발표 시각 | 0200 | 0500 | 0800 | 1100 | 1400 | 1700 | 2000 | 2300 |
| :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: | :---: |
| 호출 가능 시각 | 0215~ | 0515~ | 0815~ | 1115~ | 1415~ | 1715~ | 2015~ | 2315~ |

```
결정 알고리즘:
1. 현재 시각 = now()
2. 가장 최근의 발표 시각(base_time) 선택
3. IF (now - base_time < 15분)
   → 이전 발표 시각 사용 (데이터 미생성 상태일 수 있으므로)
4. 자정~02:15 사이 → base_date를 전날, base_time을 2300으로 설정
```

### 2.5 데이터 정제 (PCP 강수량 파싱)

기상청의 `PCP` 응답값은 **숫자가 아닌 문자열**로 반환될 수 있다.

| 응답 원본 | 변환 값 | 비고 |
| :--- | :---: | :--- |
| `"강수없음"` | `0.0` | |
| `"1mm 미만"` | `0.5` | 미세 강수 처리용 기본값 |
| `"30~50mm"` | `40.0` | 범위의 중간값 사용 |
| `"50mm 이상"` | `50.0` | 최솟값 사용 |
| `"12.5"` | `12.5` | 정상 수치 |

## 3. API-B: 종관기상관측(ASOS) 일별 조회 (과거 10년)

### 3.1 도입 배경

AI 작물 추천의 **환경 적합도(25%)**를 산출하려면, "현재 날씨"가 아닌 **"이 지역의 평년 기후가 해당 작물에 적합한가"**를 판단해야 한다.  
기후 분석의 신뢰도를 확보하기 위해 **최근 10년**의 일별 관측 데이터를 수집·분석한다.

> 💡 **왜 10년인가?**  
> - 기상청 공식 평년값은 30년 기준이나, 최근 기후 변화(지구 온난화)가 급격하여 30년 전 데이터는 현실과 괴리가 크다.  
> - 농업 현장에서는 **최근 5~10년** 데이터를 '현실적인 기후'로 사용하는 추세이다.  
> - 1~2년치만으로는 특이 기상 사례에 매몰될 수 있어 통계적 안정성이 부족하다.

### 3.2 호출 정보

| 항목 | 내용 |
| :--- | :--- |
| **Base URL** | `http://apis.data.go.kr/1360000/AsosDalyInfoService/getWthrDataList` |
| **Method** | `GET` |

### 3.3 요청 파라미터 (Request)

| 항목명 | 변수명 | 필수 | 값(예시) | 설명 |
| :--- | :--- | :---: | :--- | :--- |
| 서비스키 | `serviceKey` | Y | `(발급키)` | 공공데이터포털 Encoded 키 |
| 페이지번호 | `pageNo` | Y | `1` | |
| 한 페이지 결과 수 | `numOfRows` | Y | `999` | 최대 999건 |
| 응답형식 | `dataType` | Y | `JSON` | |
| 시작일 | `startDt` | Y | `20160101` | 10년 전 시작일 |
| 종료일 | `endDt` | Y | `20251231` | 분석 종료일 |
| 지점번호 | `stnIds` | Y | `202` | 양평 관측소 |

> 📌 **지점번호 참고:** 양평 = `202`. 서비스 전국 확대 시 지점번호 매핑 테이블 필요.

### 3.4 응답 데이터 — AI 환경 점수 산출용

| 응답 필드 | 항목명 | 단위 | AI 분석 활용 |
| :---: | :--- | :---: | :--- |
| `stnNm` | 관측소명 | — | DB `stn_name`에 저장 → AI 응답 시 지역명 표시 |
| `avgTa` | 일 평균 기온 | ℃ | 월별 10년 평균 산출 → 작물별 생육 한계 온도와 비교 |
| `minTa` | 일 최저 기온 | ℃ | **냉해 분석:** 10년간 마지막 서리일(0℃ 이하 마지막 날짜) 통계 |
| `maxTa` | 일 최고 기온 | ℃ | **고온 장해 분석:** 여름철 35℃ 이상 일수 통계 |
| `sumRn` | 일 강수량 | mm | 월별 누적 강수 패턴 → 관수 시설 필요성 판단 |
| `sumSsHr` | 일 합계 일조시간 | hr | 일조량 민감 작물(딸기, 토마토 등) 적합성 최종 판정 |
| `avgRhm` | 일 평균 습도 | % | 고습도 지역 여부 판정 → 병해충 취약 작물 제외 |
| `avgWs` | 일 평균 풍속 | m/s | 풍해 취약 작물 판단 → 방풍 시설 필요성 |

### 3.5 AI 환경 점수 산출 로직

```
환경 적합도 산출 프로세스:

1. DB에서 해당 지역의 10년 월별 통계 추출
   ├── 월별 평균 기온 (avgTa의 10년 평균)
   ├── 월별 평균 강수량 (sumRn의 10년 평균)
   ├── 월별 평균 일조시간 (sumSsHr의 10년 평균)
   └── 마지막 서리일 / 첫 서리일 (minTa ≤ 0℃ 기준)

2. 작물 적정 조건 테이블(crops.climate_conditions)과 매칭
   ├── 기온 매칭률: 작물 적정 온도 범위 내 월수 / 재배 기간 월수
   ├── 강수 매칭률: 작물 적정 강수량 범위 내 월수 / 재배 기간 월수
   ├── 일조 매칭률: 작물 최소 일조시간 충족 월수 / 재배 기간 월수
   └── 서리 안전도: 파종 예정일 > 마지막 서리일 + 여유일수(7일)

3. 환경 점수 = (기온 매칭률 × 0.4) + (강수 매칭률 × 0.2)
             + (일조 매칭률 × 0.2) + (서리 안전도 × 0.2)
   → 0~100 스케일로 정규화

4. 단기예보(API-A) 보정
   └── 현재 기온이 10년 평균 대비 ±5℃ 이상 이탈 시 
       → "올해 이상 기후 감지" 플래그 추가 (추천 근거 문장에 반영)
```

## 5. 예외 처리 및 장애 대응

### 5.1 공통 에러 코드

| ErrorCode | HTTP | 상황 | 대응 |
| :--- | :---: | :--- | :--- |
| `API_CONNECTION_FAIL` | 502 | 기상청 서버 무응답 / 타임아웃 | Retry 3회 후 Fallback |
| `API_INVALID_KEY` | 401 | 서비스키 만료 또는 미등록 | 관리자 알림 발송 |
| `API_PARAM_ERROR` | 400 | nx, ny 범위 초과 등 | 좌표 재검증 후 재시도 |
| `API_QUOTA_EXCEEDED` | 429 | 일일 호출 횟수 10,000회 초과 | 캐시 데이터로 대체 |

### 5.2 Retry 정책

```
최대 재시도: 3회
대기 전략: Exponential Backoff (1초 → 2초 → 4초)
타임아웃: 요청당 5초
```

### 5.3 Fallback 전략

| 장애 상황 | Fallback 데이터 |
| :--- | :--- |
| 단기예보 API 장애 | Redis에 저장된 마지막 성공 캐시 데이터 반환 |
| ASOS API 장애 | DB의 기존 weather_data 데이터 유지 (배치 스킵) |
| 양쪽 모두 장애 | AI 추천 시 환경 점수를 **50점(중립값)**으로 고정, 추천 근거에 "기상 데이터 일시 불가" 명시 |

### 5.4 호출 제한 관리

- 공공데이터포털 기본 쿼터: **일일 10,000회**
- 절감 전략:
  - 동일 격자 좌표 그룹핑 (인접 주소는 같은 격자로 묶임)
  - Redis 캐시 적중 시 외부 호출 생략
  - ASOS 배치는 1일 1회만 실행
