## 1. 개요

FarmBalance 시스템은 농촌진흥청이 제공하는 **흙토람 토양환경정보 서비스**를 연동하여, 유저의 농장 필지에 대한 토양 화학 성분을 자동으로 수집한다.

### 1.1 시스템 내 활용처

```
  [카카오 로컬 API]                    [흙토람 API]
       │                                   │
       │  주소 → 법정동코드 + 번지          │  PNU → 토양 성분 데이터
       │                                   │
       └────────┬──────────────────────────┘
                │
                ▼
     ┌──────────────────────────┐
     │   PNU 코드 생성 유틸리티   │
     │  (법정동 10자리 + 번지 조합) │
     └────────────┬─────────────┘
                  │
        ┌─────────┴──────────────────────────────┐
        ▼                                        ▼
  ┌──────────────┐                    ┌──────────────────────┐
  │  농장 등록    │                    │  AI 추천 엔진         │
  │  (FRM-001)   │                    │  (FRM-002)            │
  │              │                    │                      │
  │  farms 테이블 │                    │  환경 적합도(25%) 중   │
  │  soil_type   │                    │  토양 적합도(12.5%)    │
  │  자동 저장    │                    │  점수 산출             │
  └──────────────┘                    └──────────────────────┘
        │
        ▼
  ┌──────────────────────┐
  │  비료·시비 가이드 알림  │
  │  (COM-003)            │
  │                      │
  │  기상 데이터와 결합    │
  │  → 최적 시비 시점 알림 │
  └──────────────────────┘
```

| 활용처 | 기능 ID | 구체적 활용 |
| :--- | :---: | :--- |
| **AI 작물 추천** | FRM-002 | 토양 pH·유기물·인산 등이 작물 적정 조건에 맞는지 매칭 → 환경 점수 산출 |
| **농장 등록** | FRM-001 | 주소 입력 시 토양 데이터 자동 로드 → `farms.soil_type` 자동 채움 |
| **비료 처방 알림** | COM-003 | 토양 양분 부족 + 기상 조건(비 안 옴) 결합 → 시비 적기 알림 |

## 2. 데이터 획득 파이프라인 (주소 → PNU → 토양 데이터)

흙토람 API는 일반 주소가 아닌 **PNU 코드(21자리 필지 고유 번호)**를 요구한다.  
따라서 유저 주소를 PNU로 변환하는 중간 과정이 반드시 필요하다.

### 2.1 변환 흐름

```
Step 1. 유저가 농장 주소를 입력 (또는 카카오맵에서 위치 클릭)
         └── "경기도 양평군 양서면 복포리 123-4"

Step 2. 카카오 로컬 API 호출 (address → 법정동코드 변환)
         ├── API: GET https://dapi.kakao.com/v2/local/search/address
         ├── Input:  query = "경기도 양평군 양서면 복포리 123-4"
         └── Output: {
               "address": {
                 "b_code": "4183031024",    ← 법정동 코드 (10자리)
                 "main_address_no": "123",  ← 본번
                 "sub_address_no": "4",     ← 부번
                 "mountain_yn": "N"         ← 산 여부
               }
             }

Step 3. PNU 코드 조립 (백엔드 유틸리티)
         ├── 법정동 코드:  4183031024       (10자리)
         ├── 대장 구분:    1                (1: 토지, 2: 임야)
         ├── 산 여부:      1                (1: 일반, 2: 산)
         ├── 본번:         0275             (4자리, 좌측 0 패딩)
         ├── 부번:         0000             (4자리, 좌측 0 패딩)
         └── PNU:          4183033029102750000  (19자리)

Step 4. 흙토람 API 호출
         └── pnu = "4183031024110123004"
```

### 2.2 카카오 로컬 API 요청 파라미터

| 항목명 | 변수명 | 필수 | 값(예시) | 설명 |
| :--- | :--- | :---: | :--- | :--- |
| 인증키 | `Authorization` | Y | `KakaoAK {REST_KEY}` | 카카오 REST API 키 (헤더) |
| 검색어 | `query` | Y | `양평군 양서면 복포리 123-4` | 주소 텍스트 |

> 📌 카카오 로컬 API는 `전체_통합.md`에 이미 기술 스택으로 포함되어 있으므로(카카오맵 API), 별도 신규 연동이 아닌 **기존 인프라 활용**이다.

## 4. 응답 데이터 상세 (Response)

### 4.1 토양 화학 성분

| 코드 | 항목명 | 단위 | 설명 | 적정 범위 |
| :---: | :--- | :---: | :--- | :---: |
| `Ph` | 산도 | pH | 토양 산성/알칼리성 | 6.0 ~ 7.0 |
| `Om` | 유기물 함량 | g/kg | 토양 비옥도 지표 | 25 ~ 35 |
| `vldpha` | 유효인산 | mg/kg | 열매 결실 핵심 성분 | 200 ~ 300 |
| `posash` | 치환성 칼륨(K) | cmolc/kg | 뿌리 강화·병해 저항 | 0.5 ~ 0.8 |
| `poscalcium` | 치환성 칼슘(Ca) | cmolc/kg | 세포벽 형성 | 5.0 ~ 6.0 |
| `posmg` | 치환성 마그네슘(Mg) | cmolc/kg | 광합성(엽록소) 필수 | 1.5 ~ 2.0 |
| `Ec` | 전기전도도 | dS/m | 염류 농도 (과다 시 생육 장해) | < 2.0 |

### 4.2 화면 노출용 상태 매핑 테이블

전문 수치를 사용자가 이해할 수 있는 텍스트로 변환한다.

| 판정 기준 | 상태 코드 | 화면 표시 텍스트 | UI 색상 |
| :--- | :---: | :--- | :---: |
| 적정 범위 내 | `NORMAL` | "적정 — 현재 상태 양호" | 🟢 Green |
| 적정 범위 미만 | `LOW` | "부족 — 시비 권장" | 🟡 Amber |
| 적정 범위 초과 | `HIGH` | "과다 — 추비 자제" | 🔴 Red |
| 데이터 없음 | `UNKNOWN` | "토양 분석 데이터를 확인할 수 없습니다" | ⚪ Gray |

**판정 로직 (항목별):**

```
IF (측정값 >= 적정 범위 하한) AND (측정값 <= 적정 범위 상한)
    → NORMAL
ELSE IF (측정값 < 적정 범위 하한)
    → LOW
ELSE IF (측정값 > 적정 범위 상한)
    → HIGH
```

## 5. AI 환경 점수 산출 로직 (토양 부분)

### 5.1 환경 적합도 구성

`수경02_기능명세서`의 FR-FM-002에 정의된 환경 적합도(25%)는 **기상 + 토양**으로 구성된다.

```
환경 적합도 (25%) = 기상 적합도 (12.5%) + 토양 적합도 (12.5%)
```

### 5.2 토양 적합도 산출 공식

```
토양 적합도 산출 프로세스:

1. 흙토람에서 가져온 해당 필지의 토양 성분 로드
   ├── pH, OM, vldpha, posash, poscalcium, posmg, Ec

2. 작물 적정 토양 조건 테이블(crops.climate_conditions 내 soil 필드)과 매칭
   ├── pH 매칭:    |작물 적정 pH - 실제 pH| ≤ 0.5 → 100점, ≤ 1.0 → 70점, 그 외 → 30점
   ├── OM 매칭:    적정 범위 내 → 100점, LOW → 60점 (비료 보정 가능), HIGH → 50점
   ├── 인산 매칭:  적정 범위 내 → 100점, LOW → 40점 (결실 영향 큼), HIGH → 70점
   └── 양이온 매칭: 3종(K, Ca, Mg) 중 적정 개수 / 3 × 100

3. 토양 점수 = (pH 점수 × 0.35) + (OM 점수 × 0.25)
             + (인산 점수 × 0.25) + (양이온 점수 × 0.15)
   → 0~100 스케일로 정규화

4. 최종 환경 점수에 반영
   └── environment_score = (기상 점수 × 0.5) + (토양 점수 × 0.5)
```

### 5.3 작물 적정 토양 조건 (crops.climate_conditions JSONB 확장)

```json
{
  "climate": {
    "temp_min": 15, "temp_max": 30,
    "rainfall_min": 80, "rainfall_max": 200
  },
  "soil": {
    "ph_min": 6.0, "ph_max": 6.8,
    "om_min": 25, "om_max": 35,
    "p2o5_min": 200, "p2o5_max": 350,
    "k_min": 0.5, "k_max": 0.8
  }
}
```

> 📌 ERD의 `crops.climate_conditions` JSONB 컬럼에 위 `soil` 필드를 추가하면, 별도 테이블 없이 기상·토양 조건을 통합 관리할 수 있다.

## 6. 기상 + 토양 결합 알림 시나리오

단기예보(EXT-WEATHER-001)와 토양 데이터를 결합하여 **맞춤형 영농 알림**을 생성한다.

| 조건 조합 | 알림 메시지 | 알림 타입 |
| :--- | :--- | :---: |
| `토양 인산: LOW` + `내일 강수: 없음` | "토양 분석 결과 인산이 부족합니다. 내일은 비 예보가 없어 비료 주기 좋은 날입니다." | GUIDE |
| `토양 pH: LOW(산성)` + `재배 작물: 시금치(알칼리 선호)` | "현재 토양이 산성입니다. 시금치 재배를 위해 석회 시비를 권장합니다." | GUIDE |
| `토양 EC: HIGH(염류 과다)` + `재배 작물 등록됨` | "토양 염분이 높습니다. 심경(깊이갈이)으로 염류를 분산시킨 후 파종하세요." | BALANCE_WARN |
| `토양 K: LOW` + `내일 풍속: 4m/s 이상` | "토양 칼륨 부족으로 뿌리가 약할 수 있습니다. 내일 강풍 예보가 있으니 지지대를 점검하세요." | BALANCE_WARN |

```
│     │ farms 테이블  │  │ AI 추천    │  │ 알림 서비스        │     │
│     │ soil_type    │  │ 토양 점수  │  │ 기상+토양 결합     │     │

│     └──────────────┘  └────────────┘  └───────────────────┘     │
└───────────────────────────────────────────────────────────────────┘
```

### 7.2 캐싱 전략

| 대상 | 저장소 | TTL | 사유 |
| :--- | :---: | :---: | :--- |
| 토양 성분 데이터 | **Redis** | 24시간 | 토양은 단기간에 변하지 않음 |
| 주소→법정동 변환 결과 | **Redis** | 7일 | 법정동 코드는 거의 고정값 |
| PNU→토양 매핑 (DB) | **PostgreSQL** | 배치 갱신 | 장기 보관용 이력 데이터 |

## 8. 예외 처리

### 8.1 에러 코드

| ErrorCode | HTTP | 상황 | 대응 |
| :--- | :---: | :--- | :--- |
| `API_CONNECTION_FAIL` | 502 | 흙토람 서버 무응답 | Retry 3회 → Fallback |
| `SOIL_DATA_NOT_FOUND` | 200 | 해당 PNU에 토양 데이터 없음 | 읍면리 평균 데이터로 대체 |
| `PNU_CONVERT_FAIL` | 400 | 주소→PNU 변환 실패 | "정확한 지번 주소를 입력해 주세요" |
| `API_QUOTA_EXCEEDED` | 429 | 일일 호출 횟수 초과 | 캐시 데이터로 대체 |

### 8.2 Fallback 전략

| 장애 상황 | Fallback 데이터 |
| :--- | :--- |
| 흙토람 API 장애 | Redis 마지막 성공 캐시 반환 |
| 해당 필지 데이터 없음 | 해당 읍/면/리의 **평균 토양 통계** 사용 |
| 카카오 로컬 API 장애 | 유저에게 PNU 직접 입력 안내 (토지이용규제정보 서비스 링크 제공) |
| 양쪽 모두 장애 | AI 추천 시 토양 점수를 **50점(중립값)**으로 고정, 근거에 "토양 데이터 일시 불가" 명시 |

### 8.3 Retry 정책

```
최대 재시도: 3회
대기 전략: Exponential Backoff (1초 → 2초 → 4초)
타임아웃: 요청당 5초
```

## 9. 보안 및 운영

| 항목 | 정책 |
| :--- | :--- |
| **API Key 관리** | `.env` 파일 관리. 흙토람 키·카카오 키 각각 별도 변수 |
