# 🤖 Google Gemini 1.5 Flash AI 연동 기능명세서

> **문서 ID:** EXT-AI-001  
> **작성일:** 2026-04-23  
> **버전:** v1.0  
> **관련 기능:** FRM-002 (AI 작물 추천), FRM-003 (병해충 자가진단), COM-QNA (영농 챗봇)  
> **관련 ERD:** `crops`, `farms`  
> **연계 명세서:** EXT-FARM-001 (농사로), EXT-WEATHER-001 (기상청), EXT-SOIL-001 (흙토람)

## 1. 개요

Google **Gemini 1.5 Flash**는 FarmBalance 시스템의 **지능 계층(Intelligence Layer)**이다.  
다른 외부 API(기상청·흙토람·농사로·KOSIS)가 '데이터를 가져오는' 역할이라면, Gemini는 그 데이터를 **'해석하고 사람의 언어로 전달하는'** 역할을 담당한다.

### 1.1 시스템 내 활용처

| 활용처 | 기능 ID | 입력 | 출력 | 모드 |
| :--- | :---: | :--- | :--- | :---: |
| **AI 작물 추천 근거 생성** | FRM-002 | 4항목 점수 + 작물 데이터 | 추천 사유 문장 3개 이상 | Text |
| **영농 상담 챗봇** | COM-QNA | 유저 질문 텍스트 | 전문가 수준의 한국어 답변 | Text |
| **병해충 이미지 진단** | FRM-003 | 유저 촬영 사진 + 질문 | 병명·해충명 + 방제법 3가지 | **Multimodal** |
| **재배 가이드 요약** | FRM-003 | 농사로 HTML 원문 (정제 후) | 3줄 핵심 요약 | Text |
| **수급 이상 알림 해설** | COM-003 | 수급 비율 수치 + 지역 | "양평군 고추 공급이 과잉 상태입니다..." | Text |

### 1.2 다른 외부 API와의 관계

```
┌───────────────────────────────────────────────────────────┐
│  데이터 수집 계층 (Raw Data)                               │
│                                                           │
│  기상청 → 기온, 강수량, 풍속 (숫자)                        │
│  흙토람 → pH, 유기물, 인산 (숫자)                          │
│  KOSIS  → 10년 생산량 (숫자)                               │
│  농사로 → 재배법, 병해충 가이드 (HTML 텍스트)               │
└───────────────────────┬───────────────────────────────────┘
                        │
                        ▼
┌───────────────────────────────────────────────────────────┐
│  지능 계층 (Intelligence) — Gemini 1.5 Flash               │
│                                                           │
│  숫자 → "양평군의 올해 기온은 고추 재배에 적합합니다"       │
│  점수 → "수급이 부족한 작물이므로 재배를 권장합니다"        │
│  HTML → "핵심 요약: ① 정식 후 30일에 추비 ② ..."           │
│  사진 → "고추 탄저병으로 추정됩니다. 방제법: ..."           │
└───────────────────────────────────────────────────────────┘
```

## 2. API 호출 상세 (Request)

### 2.1 호출 정보

| 항목 | 내용 |
| :--- | :--- |
| **Endpoint** | `https://generativelanguage.googleapis.com/v1beta/models/gemini-1.5-flash:generateContent` |
| **Method** | `POST` |
| **Auth** | URL 파라미터 `?key={GEMINI_API_KEY}` |
| **Content-Type** | `application/json` |

### 2.2 요청 Payload 구조

```json
{
  "system_instruction": {
    "parts": [{ "text": "§3.1 시스템 지시문 참고" }]
  },
  "contents": [
    {
      "parts": [
        { "text": "유저의 질문 텍스트" },
        { "inline_data": {                    // 이미지 첨부 시에만
            "mime_type": "image/jpeg",
            "data": "(Base64 인코딩된 이미지)"
        }}
      ]
    }
  ],
  "generationConfig": {
    "temperature": 0.2,
    "maxOutputTokens": 1024,
    "responseMimeType": "application/json"    // 구조화 응답 필요 시
  },
  "safetySettings": [
    {
      "category": "HARM_CATEGORY_DANGEROUS_CONTENT",
      "threshold": "BLOCK_MEDIUM_AND_ABOVE"
    }
  ]
}
```

### 2.3 요청 파라미터 상세

| 필드 | 필수 | 권장값 | 설명 |
| :--- | :---: | :--- | :--- |
| `system_instruction` | Y | §3.1 참고 | 모델의 페르소나·행동 강령. **모든 호출에 반드시 포함** |
| `contents[].parts[].text` | Y | 유저 질문 | 텍스트 입력 |
| `contents[].parts[].inline_data` | N | Base64 이미지 | 멀티모달 입력 (이미지 진단 시) |
| `temperature` | N | `0.2` | **낮을수록 정확** — 농업 정보의 신뢰성을 위해 낮게 설정 |
| `maxOutputTokens` | N | `1024` | 답변 길이 제한. 챗봇: 1024, 요약: 512 |
| `responseMimeType` | N | `application/json` | 추천 근거 등 구조화된 응답이 필요할 때 사용 |
| `safetySettings` | N | `BLOCK_MEDIUM_AND_ABOVE` | 유해 콘텐츠 차단 수준 |

## 3. AI 페르소나 및 프롬프트 설계

### 3.1 시스템 지시문 (System Instruction)

모든 Gemini 호출에 반드시 포함되는 **모델의 정체성 설정**.

```
너는 양평군 농업기술센터 소속의 20년 경력 베테랑 농업 컨설턴트야.

[행동 강령]
1. 모든 답변은 친절하고 전문적인 한국어로 작성해.
2. 농업 전문 용어(예: 추비, 정식, 관수)는 초보 농민도 이해할 수 있도록 
   괄호 안에 쉬운 설명을 덧붙여줘. (예: "추비(웃거름)")
3. 답변의 근거가 되는 데이터(기온, 토양 성분 등)가 있으면 함께 언급해줘.
4. 농업과 관련 없는 질문에는 "죄송합니다. 저는 농업 관련 질문에만 
   답변할 수 있습니다."라고 정중히 거절해.
5. 답변 끝에는 반드시 다음 문구를 붙여:
   "⚠️ 본 답변은 AI의 분석 결과이며, 정확한 진단을 위해 
   가까운 농업기술센터 또는 전문가와 상의하세요."
```

### 3.2 기능별 프롬프트 템플릿

#### 3.2.1 작물 추천 근거 생성 (FRM-002)

FRM-002의 처리 로직 §6에서 "추천 근거 문장 생성(작물별 3개 이상)"에 Gemini를 활용한다.

```
[Context]
추천 작물: {crop_name}
수급 점수: {balance_score}점 (현재 상태: {balance_status})
환경 점수: {environment_score}점 (기온: {temp}°C, 토양 pH: {ph})
수익 점수: {profit_score}점 (3년 평균 가격: {avg_price}원/kg)
역량 점수: {capability_score}점 (재배 경험: {has_experience})

[요청]
위 데이터를 바탕으로 이 작물을 추천하는 이유를 
농민이 바로 이해할 수 있는 한국어 문장 3개로 작성해줘.
각 문장은 가장 높은 점수 항목 순서대로 배치해줘.
```

**JSON 응답 스키마 지정:**

```json
{
  "reasons": [
    "현재 양평군에서 고추 공급이 부족한 상태입니다. 재배하시면 높은 수요를 기대할 수 있습니다.",
    "최근 3년 평균 가격이 kg당 12,000원으로 수익성이 상위권에 해당합니다.",
    "기존에 고추 재배 경험이 있어 안정적인 생산이 가능합니다."
  ]
}
```

#### 3.2.2 병해충 이미지 진단 (FRM-003 — Multimodal)

```
[이미지 첨부됨]
이 이미지는 사용자의 농장에서 촬영한 작물 사진이야.

다음 형식으로 분석해줘:
1. 작물 종류: (판별된 작물명)
2. 증상 분석: (잎·줄기·과실의 이상 증상 서술)
3. 예상 병명: (가장 유력한 병명 또는 해충명)
4. 확신도: (높음/중간/낮음)
5. 즉시 조치 방법: (3가지 리스트)
6. 장기 방제 전략: (1~2문장)
```

#### 3.2.3 재배 가이드 요약 (농사로 연계)

```
다음은 농촌진흥청에서 제공하는 '{crop_name}' 재배 기술 문서야.
이 내용을 초보 농민이 바로 실천할 수 있도록 
핵심 3가지 포인트로 요약해줘. 각 포인트는 2줄 이내로.

[원문]
{stripped_html_content}
```

## 4. 멀티모달 처리 (이미지)

### 4.1 이미지 전송 파이프라인

```
유저가 사진 촬영/업로드
    │
    ▼
Next.js (프론트엔드)
    ├── 이미지 파일 수신
    ├── 파일 크기 검증 (최대 10MB)
    └── BFF API Route → Spring Boot 전송
            │
            ▼
Spring Boot (백엔드)
    ├── 이미지 리사이징 (최대 1024×1024)
    ├── Base64 인코딩
    ├── 프롬프트 템플릿(§3.2.2) + 이미지 결합
    └── Gemini API 호출
            │
            ▼
Gemini 응답 수신
    ├── 구조화된 진단 결과 파싱
    ├── 확신도가 "낮음"이면 전문가 상담 강력 권유 문구 추가
    └── 프론트엔드로 반환
```

### 4.2 이미지 제한 사항

| 항목 | 제한값 | 사유 |
| :--- | :--- | :--- |
| 최대 파일 크기 | 10MB | Gemini API 제한 |
| 권장 해상도 | 1024×1024 이하 | 토큰 소비 최적화 |
| 지원 형식 | JPEG, PNG, WebP | Gemini 지원 포맷 |
| 1회 최대 이미지 수 | 1장 | 진단 정확도 확보 (단일 증상 집중) |

## 5. 토큰 관리 및 비용 최적화

### 5.1 Free Tier 쿼터 (2026년 기준)

| 항목 | 제한 |
| :--- | :--- |
| 분당 요청 수 (RPM) | 15회 |
| 일일 요청 수 (RPD) | 1,500회 |
| 분당 토큰 수 (TPM) | 1,000,000 |

### 5.2 기능별 토큰 소비 예상

| 기능 | 입력 토큰 (예상) | 출력 토큰 (예상) | 1일 예상 호출 |
| :--- | :---: | :---: | :---: |
| 추천 근거 생성 | ~500 | ~300 | ~100회 |
| 챗봇 대화 | ~200 | ~500 | ~500회 |
| 이미지 진단 | ~1,000 (이미지 포함) | ~400 | ~50회 |
| 가이드 요약 | ~2,000 (원문 포함) | ~200 | ~100회 |
| **합계** | — | — | **~750회/일** |

> 📌 Free Tier 일일 한도 1,500회 이내에서 충분히 운영 가능.  
> 단, **토큰 절약을 위해** 가이드 요약 시 원문은 3,000자 이내로 잘라서 전송한다.

### 5.3 토큰 절약 전략

| 전략 | 대상 | 설명 |
| :--- | :--- | :--- |
| **응답 캐싱** | 추천 근거 | 동일 작물+동일 점수대 → Redis 캐시 (TTL 1시간) |
| **입력 트리밍** | 가이드 요약 | 농사로 HTML → 태그 제거 후 **3,000자 이내** 절삭 |
| **이미지 리사이징** | 병해충 진단 | 1024×1024 이하 축소 → 이미지 토큰 절감 |
| **Rate Limiting** | 챗봇 | 유저당 분당 3회 호출 제한 → 남용 방지 |

## 6. 예외 처리

### 6.1 에러 코드

| ErrorCode | HTTP | 상황 | 대응 |
| :--- | :---: | :--- | :--- |
| `AI_GENERATION_ERROR` | 500 | Gemini 응답 생성 실패 | Retry 2회 → "AI 서비스 일시 장애" 안내 |
| `AI_SAFETY_BLOCKED` | 200 | Safety Settings에 의해 차단 | "부적절한 요청이 감지되었습니다" |
| `AI_QUOTA_EXCEEDED` | 429 | RPM/RPD 초과 | "잠시 후 다시 시도해 주세요" + 대기열 안내 |
| `AI_TIMEOUT` | 503 | 응답 2초 초과 | 캐시된 이전 유사 응답 제공 또는 템플릿 문장 Fallback |
| `IMAGE_TOO_LARGE` | 413 | 이미지 10MB 초과 | "이미지 크기를 줄여서 다시 업로드해 주세요" |

### 6.2 Fallback 전략

| 장애 상황 | Fallback |
| :--- | :--- |
| Gemini API 다운 | 추천 근거 → **템플릿 기반 문장 자동 조합** (§3.2.1의 정적 버전) |
| 이미지 진단 실패 | 농사로 API의 병해충 사진 목록으로 **수동 대조** 안내 |
| 가이드 요약 실패 | 농사로 원문 HTML을 **그대로 렌더링** (요약 없이) |
| Safety 차단 | "농업 관련 질문으로 다시 시도해 주세요" 안내 |

### 6.3 Retry 정책

```
최대 재시도: 2회 (AI 응답은 비결정적이므로 3회 이상 불필요)
대기 전략: Fixed Delay 1초
타임아웃: 요청당 10초 (이미지 포함 시 15초)
```

## 7. 법적 고지 및 UI 필수 요소

### 7.1 할루시네이션(환각) 대응

AI가 사실과 다른 정보를 진실처럼 답변할 수 있으므로, **모든 AI 응답 화면에 아래 요소를 필수 표시**한다.

| UI 요소 | 위치 | 표시 내용 |
| :--- | :---: | :--- |
| **면책 문구** | 응답 하단 (항상) | "⚠️ AI의 답변은 참고용이며, 실제 농업 현장과는 차이가 있을 수 있습니다. 정확한 진단을 위해 가까운 농업기술센터와 상의하세요." |
| **모델 출처** | 응답 하단 (항상) | "Powered by Google Gemini" |
| **확신도 뱃지** | 병해충 진단 결과 옆 | 🟢 높음 / 🟡 중간 / 🔴 낮음 — 낮음일 때 "전문가 상담 강력 권장" |
| **피드백 버튼** | 응답 하단 | 👍 / 👎 — 답변 품질 피드백 수집 (향후 프롬프트 개선용) |

### 7.2 데이터 프라이버시

| 항목 | 정책 |
| :--- | :--- |
| 유저 입력 데이터 | Google AI API 이용약관에 따라 모델 학습에 활용될 수 있음을 **이용약관에 명시** |
| 개인정보 마스킹 | 유저 이름·전화번호 등이 질문에 포함될 경우, 백엔드에서 **마스킹 후** Gemini에 전송 |
| 대화 이력 저장 | 품질 개선 목적으로 DB에 저장 / 유저 요청 시 삭제 가능 |

## 9. 보안 및 운영

| 항목 | 정책 |
| :--- | :--- |
| **API Key 관리** | `.env` → `GEMINI_API_KEY`. GitHub 노출 시 즉시 키 정지됨 |

| **비용 모니터링** | 관리자 대시보드(ADM-001)에서 일별 토큰 소비량·호출 횟수 모니터링 |
