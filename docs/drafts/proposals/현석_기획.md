## 1. 프로젝트 개요

### 1.1 배경 및 문제 정의

양평 지역 농민들은 **작물 재배 선택 시 정보 비대칭** 문제를 겪고 있다.

| 문제 | 설명 |
|------|------|
| **쏠림 현상** | 특정 작물(예: 양파)이 유행하면 다수의 농가가 동시에 재배 → 공급 과잉 → 가격 폭락 |
| **반복적 손실** | 폭락을 경험한 농민이 다음 해 작물을 변경 → 해당 작물 공급 부족 → 가격 상승 → 악순환 반복 |
| **커뮤니케이션 부재** | 농민 간 재배 계획을 공유할 채널이 없어 수급 조절 불가 |
| **중간 조정자 부재** | 실시간으로 작물 수급 상황을 모니터링하고 알려줄 주체가 없음 |

### 1.2 프로젝트 목표

1. **작물 밸런스 시각화** — 지역 내 작물별 재배 현황을 실시간으로 보여줌
2. **AI 기반 작물 추천** — 토양·기후·시세 데이터를 활용한 RAG 기반 작물 추천
3. **농민 간 소통 채널** — 커뮤니티(게시판)를 통한 정보 교류
4. **소규모 직거래 상점** — 농산물 소규모 판매·구매 기능
5. **데이터 기반 의사결정 지원** — 지자체·관리자를 위한 대시보드

## 2. 역할자(Actor) 정의

| 역할 | 설명 | 가입 흐름 |
|------|------|-----------|
| **일반 유저** | 회원가입 시 기본 역할. 커뮤니티 참여, 상점 구매, 농부 등록 신청 가능 | 회원가입 → 일반유저 |

## 3. 도메인별 기능 상세

### 3.1 🧑‍🌾 농부 도메인

#### 3.1.1 개인 농장 관리
- 농장 등록: 위치, 토지 면적, 토양 타입
- 농장 정보 수정/조회
- 재배 이력 관리

#### 3.1.2 종자 등록
- 구매한 종자 등록 (종자명, 수량, 구매일, 구매처)
- 파종 계획 등록 (작물명, 파종일, 목표 재배량, 재배 면적)
- 수확 실적 등록 (실제 수확량, 수확일)

#### 3.1.3 작물 밸런스 조회
- 지역 내 작물별 재배 비율 시각화 (차트)
- 전년 대비 변화량 표시
- 가격·수확량 추이 그래프

#### 3.1.4 작물 추천 (AI/RAG)
- 입력: 토양 정보, 토지 면적, 기후 조건
- 출력: 추천 작물 목록 + 추천 점수 + 추천 사유
- 참고 데이터: 시세, 유통량, 판매량, 기후, 토양

#### 3.1.5 가게 정보 조회
- 카카오맵 API 연동 — 인근 농자재 가게 목록
- 가게 위치, 연락처, 취급 품목 표시

#### 3.1.6 상점 (판매)
- 농산물 상품 등록 (상품명, 가격, 수량, 설명, 이미지)
- 주문 접수 및 관리
- 판매 내역 조회

### 3.2 👤 일반유저 도메인

#### 3.2.1 농부 등록 신청
- 농장 정보 입력 → 관리자 승인 → 농부 역할 부여

#### 3.2.2 상점 (구매)
- 상품 목록 조회 / 검색
- 장바구니
- 주문 / 결제
- 주문 내역 조회

#### 3.2.3 커뮤니티
- 게시글 CRUD
- 댓글
- 카테고리별 게시판 (자유, 정보공유, 질문 등)

### 3.3 🏛️ 지자체 도메인

#### 3.3.1 대시보드 (조회 전용)
- 작물별 재배 현황 및 비율
- 농민 수, 농장 수, 총 토지 면적
- 작물별 가격 추이
- 기후·토양 종합 데이터
- 작물 로스율, 선호도 통계

### 3.4 🔧 관리자 도메인

#### 3.4.1 RAG 데이터 관리
- 외부 API 데이터 수집 설정 (기후, 토양, 유통/판매량, 가격)
- 데이터 갱신 주기 설정
- 데이터 품질 모니터링

#### 3.4.2 유저 관리
- 회원 목록 조회 / 검색
- 역할 변경 (일반유저 ↔ 농부)
- 농부 등록 승인/반려
- 계정 정지/삭제

#### 3.4.3 가게 정보 관리
- 가게 등록/수정/삭제
- 카카오맵 데이터 동기화

#### 3.4.4 커뮤니티 관리
- 게시글/댓글 삭제 (신고 처리)
- 공지사항 관리

#### 3.4.5 상점 관리
- 상품 승인/반려
- 부적합 상품 비활성화
- 주문 분쟁 처리

#### 3.4.6 대시보드 (전체)
- 지자체 대시보드 전체 기능 포함
- 추가: 씨앗 구매량, 작물 로스율, 작물별 선호도, 재배 계획 수립 현황

## 4. RAG 데이터 구성

### 4.1 내부 데이터 (사용자 입력)

| 데이터 | 수집 방법 | 설명 |
|--------|-----------|------|
| 종자 소지 데이터 | 농부가 직접 등록 | 구매 종자, 수량, 파종 계획 |
| 농부 및 농장 데이터 | 농부 등록 시 입력 | 위치, 면적, 토양 타입, 재배 이력 |
| 수확 실적 | 농부가 직접 입력 | 실제 수확량, 품질 정보 |

### 4.2 외부 데이터 (API 연동)

| 데이터 | 출처 (예정) | 비고 |
|--------|------------|------|
| 기후 정보 | 기상청 API | 기온, 강수량, 일조량 등 |
| 토양 정보 | 흙토람 API | 토양 성분, pH, 유기물 함량 |
| 농산물 유통량 | 농넷(aT) 또는 KAMIS | 도매시장 유통 데이터 |
| 농산물 판매량 | KAMIS API | 품목별 거래량 |
| 농산물 가격 | KAMIS API | 도·소매 가격 추이 |

> **참고**: 초기 개발 시에는 더미 데이터로 진행하고, 이후 실제 API로 전환 예정

## 5. 핵심 유저 플로우

### 5.1 농부의 작물 선택 플로우

```
[로그인] → [내 농장 조회] → [작물 밸런스 확인]
                                    ↓
                          지역 내 양파 재배 비율 70% 확인
                                    ↓
                         [AI 작물 추천 요청]
                                    ↓
                    토양·기후·시세 분석 → "감자" 추천 (점수 87)
                                    ↓
                         [종자 등록 → 파종 계획 수립]
                                    ↓
                         [인근 가게에서 감자 종자 구매]
```

### 5.2 일반유저 → 농부 전환 플로우

```
[회원가입 (일반유저)] → [농부 등록 신청]
                              ↓
                    농장 정보 입력 (위치, 면적, 토양)
                              ↓
                      [관리자 승인 대기]
                              ↓
                    승인 완료 → 역할 "농부"로 변경
                              ↓
                    농부 전용 기능 활성화
```

## 9. 보상 체계 (향후 검토)

농부들이 종자/재배 데이터를 적극적으로 등록하도록 유도하기 위한 보상 체계:

- 데이터 등록 시 **포인트 적립**
- 포인트로 **상점 할인 쿠폰** 교환
- 활동 우수 농부 **배지** 부여
- 지자체 연계 **보조금 우선 대상** 추천

> ⚠️ 보상 체계는 MVP 이후 2차 개발에서 상세 설계 예정

## 11. Q&A 정리 (회의 내용 반영)

| 질문 | 답변 | 기획 반영 |
|------|------|-----------|
| 씨앗을 사서 하나요? | 대부분 구매, 미사용 케이스도 존재 | 종자 등록 시 "구매" 상태만 추적 (구매 ≈ 파종 가능) |
| 데이터 출처는? | 초기 더미 → 추후 지자체 연계 | Phase 1 더미, Phase 3 실 API 연동 |
| 종자 등록 UI? | 보상체계 있으면 농민 참여 유도 가능 | 포인트 적립 기반 보상 체계 설계 |
