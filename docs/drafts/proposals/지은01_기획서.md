## 1. 프로젝트 개요

### 1.1 배경 및 문제 정의

| 문제 | 설명 |
|------|------|
| **거미집 이론(Cobweb Theory)** | 특정 작물에 농민이 쏠리면 → 공급 과잉 → 가격 폭락 → 이듬해 재배 급감 → 공급 부족의 악순환 |
| **정보 비대칭** | 초보·소규모 농업인은 주변 농가의 파종 현황, 시장 수급 정보에 접근하기 어려움 |
| **판로 부재** | 소량 생산 농가는 대형 유통망 진입이 불가능하여 수확물 폐기 빈도가 높음 |
| **데이터 부재** | 지자체는 실시간 파종 데이터가 없어 선제적 수급 관리가 불가능 |

### 1.2 솔루션: FarmBalance

**농민이 씨앗 구매 정보를 등록하면**, 지역의 실시간 파종 밀도를 계산하여 **AI가 최적 작물을 추천**하고, 지자체는 **수급 컨트롤 타워**로 활용하는 데이터 기반 상생 플랫폼.

### 1.3 핵심 가치

```
[🌱 농민] --씨앗 등록--> [📊 FarmBalance] --AI 추천 & 보상--> [🌱 농민]
                              |
                         수급 데이터
                              |
                        [🏛️ 지자체] --보조금·정책--> [🌱 농민]
                              |
                    신생 가게 매칭
                              |
                        [🏪 종자 판매점] --공동구매 혜택--> [🌱 농민]
```

## 3. 서비스 구조

### 3.1 농민용 서비스: "똑똑한 농사 길잡이"

> 농민에게 가장 중요한 것은 **'신뢰'**와 **'보상'**

#### A. 메인 대시보드 (작물 밸런스)

- 양평 지역 작물별 재배 비중 **인포그래픽** 시각화
- **핵심 데이터**: 실시간 파종 현황, 예상 출하량, 전년 대비 가격 추이
- "최대한 적게 입력하고, 명확하게 본다" — 날씨+작물 상태 카드 UI

#### B. AI 작물 추천 서비스

| 기능 | 설명 |
|------|------|
| 토양 & 기후 분석 | 흙토람 API + 기상청 데이터 결합 → 내 땅 최적 작물 점수 |
| 수급 최적화 알림 | "양파 파종량 과다 → 감자 전환 시 수익 n%↑" 동적 가이드 |
| AI 프롬프트 | Gemini 1.5 Flash 기반 — 토지면적, 토질, 주변 파종량 컨텍스트 활용 |

#### C. 종자 구매 등록

- **행동 유도**: "무슨 씨앗을 사셨나요?" 직관적 등록 화면
- **보상 체계**: 등록 시 지역 신생 가게 할인 쿠폰 / 지자체 보조금 포인트 연동
- **영수증 인증**: 씨앗 구매 영수증 사진 업로드 → 지역 화폐 포인트 지급

#### D. 직거래 셀러 허브

- 스마트폰 사진 한 장으로 간편 상품 등록
- 소규모 셀러용 주문/결제 대시보드 + 장바구니
- 찜 & 단골 관리 기능

#### E. 커뮤니티 & 로컬 마켓

- 농민 간 노하우 공유 및 품앗이 게시판
- 질문-답변 채택 시스템 (숙련 농부·지자체 답변)
- 카카오맵 연동 가게 정보 + "초보 친화도" 리뷰

### 3.2 관리자(지자체)용 서비스: "수급 컨트롤 타워"

> 지자체는 데이터를 **'조회'**하고 정책적 **'개입'**을 결정하는 곳

| 기능 | 설명 |
|------|------|
| 통합 관제 모니터링 | 양평군 전체 농장 수, 총 재배 면적, 종합 데이터 시각화 |
| 로스율·선호도 분석 | 작물별 폐기 비율, 농민 선호 작물 통계 |
| 자동 재배 계획 가이드 | 작물 쏠림 감지 → 해당 농가 일괄 '작물 변경 권고' 전송 |
| 신생 가게 협력 관리 | 가게 등록, 공동 구매 혜택 매칭, 지도 상단 노출 관리 |
| 보조금·교육 매뉴얼 배포 | 신규 농업인 대상 정부 보조금 정보 원클릭 전달 |

## 4. 비즈니스 모델 및 보상 체계

### 4.1 보상 순환 구조

```
농민 씨앗 구매 등록
    ├── 지역 파종 데이터 축적 → AI 추천 정확도 향상 → 농민 수익 증가
    └── 보상 지급
         ├── 지역 화폐 포인트
         ├── 신생 가게 할인 쿠폰 ──→ 지역 경제 활성화
         └── 지자체 보조금 연동
```

### 4.2 신생 가게 제휴 모델

- 카카오맵 API로 기존 가게 정보 수집
- **제휴 신생 가게**는 지도에서 별도 아이콘 + 상단 노출
- 농민 공동 구매 매칭 → 가게 매출 증가 → 플랫폼 수수료 모델

### 4.3 지자체 설득 논리

> "농민이 씨앗 구매 영수증을 사진 찍어 올리면 지역 화폐 포인트를 지급합니다.
> 이를 통해 실시간 파종 데이터를 확보하고, 수급 과잉을 사전에 방지할 수 있습니다."

## 5. 기술 아키텍처

### 5.1 전체 시스템 구성

```
[IoT Layer]
  MQTT 브로커(Mosquitto) ← 더미 데이터 발생기

[Backend Layer]
  Spring Boot (Hexagonal) ←→ InfluxDB (시계열)
                          ←→ PostgreSQL (영속성)

[AI/Data Layer]
  FastAPI (Python) ←→ Scikit-learn (수급 예측)
                   ←→ Gemini API (작물 추천)

[Frontend Layer]
  농민용 UI (Next.js + Tailwind CSS)
  관리자 대시보드 (Next.js + Leaflet.js)

[External APIs]
  기상청 API / 흙토람 API / 카카오맵 API
```

**데이터 파이프라인**: `IoT(MQTT) → Spring Boot → InfluxDB → FastAPI`

### 5.2 핵심 기술 스택

| 계층 | 기술 | 용도 |
|------|------|------|
| Backend 핵심 | Java, Spring Boot, JPA | 도메인 로직, REST API, 헥사고날 아키텍처 |
| 시계열 DB | InfluxDB | IoT 센서 데이터, 시간별 파종 현황 저장 |
| 영속성 DB | PostgreSQL | 사용자, 주문, 커뮤니티 등 관계형 데이터 |
| AI/Data | Python, FastAPI, Scikit-learn | 수급 예측 모델, 거미집 이론 방지 알고리즘 |
| AI 추천 | Gemini 1.5 Flash | 맞춤형 작물 추천 프롬프트 |
| Frontend | Next.js (App Router) | SSR, SEO 최적화 |
| 지도 | Leaflet.js + 카카오맵 API | 관리자 지도 시각화, 가게 정보 |
| IoT | MQTT (Mosquitto) | 센서 데이터 수집 파이프라인 |
| Infra | AWS EC2, Docker | 서버 배포, 컨테이너화 |

### 5.3 헥사고날 아키텍처 (Backend)

| Layer | 구성 |
|-------|------|
| **Domain** | `Crop`, `SeedInventory`, `MarketBalance` 등 핵심 비즈니스 로직 |
| **Application Ports** | `CropRecommendationPort`, `OrderProcessingPort` |
| **Infrastructure Adapters** | `PostgresAdapter`, `ExternalApiAdapter`, `AiAgentAdapter` |
