## 초보 및 소규모 농업인을 위한 데이터 기반 상생 플랫폼

---

## 0. 기술 아키텍처 및 표준 (Rule-based)

본 프로젝트는 `프로젝트_초기_세팅.md` 지침에 따라 다음의 기술 스택 및 아키텍처 표준을 준수합니다.

### 0.1 기술 스택
- **Backend:** Java 21, Spring Boot 3.3.x, JPA(Hibernate 6.x), PostgreSQL 16, Querydsl, jjwt 0.12.6, SpringDoc OpenAPI 2.6.0
- **Frontend:** Next.js 15 (App Router), React 19, TypeScript 5, TailwindCSS 4, shadcn/ui, framer-motion
- **Infrastructure:** Docker Compose v2, BFF(Backend For Frontend) Proxy Architecture

### 0.2 아키텍처 원칙
1. **Hexagonal Architecture (Backend):** 
   - 각 도메인은 `adapter/in/web`, `application/port`, `application/service`, `adapter/out/persistence` 구조를 가짐.
   - 비즈니스 로직은 외부 기술(DB, Web)에 의존하지 않고 독립적으로 유지.
2. **BFF (Backend For Frontend):**
   - Next.js의 API Route를 Proxy로 활용하여 보안 강화 및 데이터 정제.
   - JWT 토큰은 브라우저에 노출되지 않도록 `httpOnly` 쿠키로 BFF 계층에서 관리.
3. **API 표준:**
   - CamelCase 사용 (JSON).
   - 모든 응답은 `ApiResponse<T>` 포맷으로 통일.

## 2. 공통 기능 (Common Domain)

### 2.1 인증 및 보안 (Auth)
| 항목 | 내용 | API Endpoint |
|:---|:---|:---|
| **기능 ID** | COM-001 | - |
| **로그인/회원가입** | 이메일/소셜(카카오) 연동. JWT 발급 | `POST /api/auth/login`, `POST /api/auth/register` |
| **보안 흐름** | BFF에서 로그인 요청 → 백엔드 JWT 생성 → BFF에서 `httpOnly` 쿠키 설정 → 이후 요청은 BFF가 쿠키를 Header(Bearer)로 변환 | - |
| **예외 처리** | `INVALID_CREDENTIALS(401)`, `DUPLICATE_LOGIN_ID(409)` | - |

### 2.2 알림 시스템 (Notification)
| 항목 | 내용 | API Endpoint |
|:---|:---|:---|
| **기능 ID** | COM-003 | `GET /api/notifications` |
| **설명** | 작물 밸런스 경고(과잉/부족), 주문 알림, 공지사항 실시간/리스트 제공 | `PATCH /api/notifications/{id}/read` |

```
backend/src/main/java/com/farmbalance/
├── global/                    # Security, JWT, Error, ApiResponse, BaseEntity
```

```
frontend/src/
├── app/
│   ├── (auth)/                # login, register
│   ├── (farmer)/              # dashboard, ai-recommend, seeds
│   ├── (admin)/               # monitoring, stats
│   ├── (gov)/                 # local-admin, notice
│   ├── (shop)/                # marketplace, products
│   └── api/bff/[...path]/     # JWT Cookie <-> Header Proxy Handler
├── components/                # ui(shadcn), layout, shared
└── lib/                       # apiFetch(Wrapper), constants
```

## 8. 외부 API 및 AI 연동 명세

| API | 용도 | 캐싱 전략 (Rules 기반) |
|:---|:---|:---|
| **기상청 API** | 단기/장기 기후 예측 | 3시간 TTL (Redis/Local Cache) |
| **KAMIS** | 실시간 농산물 가격 | 1시간 TTL |
| **Gemini 1.5 Flash** | AI 추천 및 리포트 생성 | No Cache (실시간성 강조) |
| **토양환경정보** | 필지별 토양 상태 조회 | 24시간 TTL |

## 9. 에러 코드 정의 (ErrorCode)

| ErrorCode | Status | Description |
|:---|:---|:---|
| **INVALID_CREDENTIALS** | 401 | 로그인 정보 불일치 |
| **BALANCE_OVERLOAD** | 409 | 특정 지역 작물 파종 과잉 상태 |
| **API_CONNECTION_FAIL** | 502 | 외부 API(기상청 등) 연동 실패 |
| **AI_GENERATION_ERROR** | 500 | Gemini AI 응답 생성 실패 |
| **NOT_FOUND** | 404 | 요청한 리소스를 찾을 수 없음 |

## 10. 비기능 요구사항 및 보안

1. **로그 추적:** 모든 요청에 `X-Correlation-Id`를 부여하여 AI 추천부터 등록까지의 흐름 추적.
2. **성능:** Next.js stand-alone 빌드 및 SSR 활용으로 초기 LCP 2.5s 이내 달성.
3. **보안:** 
   - 비밀번호 BCrypt 암호화.
   - JWT Access Token(30분), Refresh Token(7일) 관리.
   - 모든 외부 API 통신은 백엔드 Infra 계층에서 캡슐화.
4. **환경 설정:** `docker-compose.yml`을 통한 DB, Redis, App의 원클릭 배포 환경 지원.
