> **버전**: v1.0  
> **작성일**: 2026-04-22  
> **기반**: 수경(처리로직) + 지윤(아키텍처) + 지은(ERD/API) + 현석(API/라이프사이클)

---

## 1. 역할 및 인증 체계

### 역할 정의

| 역할 | 코드 | 설명 |
|------|------|------|
| 일반유저 | `ROLE_USER` | 회원가입 시 기본. 커뮤니티·상점 구매·밸런스 조회 가능 |
| 농부 | `ROLE_FARMER` | 농장 등록 승인 후 승격. 종자 등록·AI 추천·상점 판매 추가 |
| 관리자 | `ROLE_ADMIN` | 시스템 전체 관리 |
| 지자체 | `ROLE_GOV` | 수급 대시보드·보고서 조회 전용 |

### 역할 승격 플로우

```
회원가입(ROLE_USER) → 농장 등록 신청 → 관리자 승인 → ROLE_FARMER 부여
```

## 5. 작물 밸런스 시스템

### FRM-011 작물 밸런스 조회

| 항목 | 내용 |
|------|------|
| **작물 비율** | `GET /api/balance/crop-ratio` |
| **연도별 추이** | `GET /api/balance/yearly-trend` |
| **전년 대비** | `GET /api/balance/year-comparison` |
| **접근** | ROLE_USER 이상 |
| **파라미터** | regionCode, year, season, cropId(선택) |
| **출력** | cropName, supplyRatio, balanceStatus, 게이지 차트 데이터 |
| **캐싱** | TTL 5분 (Redis) |

### 수급 상태 5단계 (Enum: BalanceStatus)

| 상태 | 코드 | 수급비율 | 의미 |
|------|------|:--------:|------|
| 과잉경고 | `EXCESS_WARNING` | > 150% | 즉시 대안 추천 |
| 과잉주의 | `EXCESS_CAUTION` | 120~150% | 주의 알림 |
| 적정 | `BALANCED` | 80~120% | 균형 |
| 부족주의 | `SHORTAGE_CAUTION` | 50~80% | 재배 권장 |
| 부족경고 | `SHORTAGE_WARNING` | < 50% | 적극 권장 |

### 수급 계산 공식

```
공급 예측 = Σ(등록면적 × 작물별 단위수확량) + 외부공급추정
수요 예측 = 지역인구 × 1인당소비량 × 계절계수
수급 비율 = 공급예측 / 수요예측 × 100
```

### 밸런스 반영 흐름

```
종자 등록 이벤트 → 수급 엔진 (5분 이내 갱신) → BalanceData 업데이트
  → 파종밀도 120% 초과 시 → 쏠림 알림 (Notification)
```

## 6. AI 작물 추천

### FRM-012 AI 작물 추천 요청

| 항목 | 내용 |
|------|------|
| **API** | `POST /api/recommend` |
| **접근** | ROLE_USER 이상 |
| **입력** | farmId (자동으로 지역·면적·토양 반영), priorityPreference(선택) |
| **처리** | ① 농장 정보 조회 ② 수급·기후·시세 데이터 수집 ③ Gemini API 호출 ④ 점수 산정 ⑤ Top 5 정렬 |
| **출력** | 작물별 { cropName, totalScore, balanceScore, envScore, profitScore, capabilityScore, reasons[3], estimatedRevenue } |
| **캐싱** | TTL 1시간 (동일 조건) |

### 점수 산정 공식

```
총점 = (수급적합도 × 0.35) + (환경적합도 × 0.25) + (수익성 × 0.25) + (농가역량 × 0.15)
```

## 13. API 공통 규격

### 응답 포맷

```json
{
  "success": true,
  "data": { ... },
  "error": null,
  "meta": { "page": 1, "size": 20, "total": 100 }
}
```

### 에러 응답

```json
{
  "success": false,
  "data": null,
  "error": { "code": "E-FRM-004-001", "message": "농장이 승인되지 않았습니다." },
  "meta": null
}
```

### API 규칙
- 네이밍: camelCase
- 헤더: `X-Correlation-Id` (요청 추적)
- 인증: BFF → JWT Header (`Authorization: Bearer {token}`)
- 페이징: `?page=0&size=20&sort=createdAt,desc`

### 에러 코드 체계

| 코드 | 설명 |
|------|------|
| `E-COM-001-xxx` | 회원가입 관련 |
| `E-COM-002-xxx` | 로그인 관련 |
| `E-FRM-004-xxx` | 종자 등록 관련 |
| `E-FRM-007-xxx` | 파종 계획 관련 |
| `E-MKT-xxx-xxx` | 상점 관련 |
| `E-GOV-xxx-xxx` | 지자체 관련 |
| `E-ADM-xxx-xxx` | 관리자 관련 |

## 14. 비기능 요구사항

| 항목 | 요구사항 |
|------|----------|
| 성능 | API 응답 500ms 이내, 밸런스 반영 5분 이내 |

| 추적 | `X-Correlation-Id` 요청 추적 |
